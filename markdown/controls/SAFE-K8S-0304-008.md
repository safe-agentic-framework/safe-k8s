# SAFE-K8S-0304-008 - Training scratch volume limits and tmpfs handling

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.4 - Secure Defaults and Resource Constraints
- Maturity: Foundational
- Classification: ai-specific

## Statement

AI training workloads must enforce emptyDir sizeLimit constraints on scratch volumes, must apply namespace ephemeral-storage quotas for training workloads, and must use tmpfs-backed emptyDir volumes for sensitive intermediate data so that temporary training data is bounded, memory-backed where required, and cleaned with pod lifecycle termination.

## Objective

Prevent disk exhaustion and sensitive intermediate-data exposure from AI training scratch space by enforcing bounded ephemeral volume usage and memory-backed handling for the most sensitive temporary data.

## Rationale

Training jobs generate large intermediate datasets and scratch artifacts that can exhaust node storage or persist on node filesystems if left on disk-backed ephemeral volumes. emptyDir size limits, namespace ephemeral-storage quotas, and tmpfs-backed handling for sensitive data reduce both availability and confidentiality risk.

## Scope

Applies to AI and ML training workloads that use emptyDir or equivalent ephemeral scratch space for staging data, tokenized samples, gradients, caches, or intermediate outputs. Inference workloads using comparable ephemeral scratch space should follow the same bounded-volume patterns where applicable.

## Enforcement

- Primary Enforcement Point: Pod specification validation and namespace quota enforcement for ephemeral scratch volumes and emptyDir media selection.
- Implementation Surface: Pod spec emptyDir configuration, emptyDir.sizeLimit enforcement, emptyDir.medium=Memory for sensitive scratch data, and namespace ResourceQuota or policy rules governing ephemeral-storage consumption in AI workload namespaces.
- Owner Primary: AI/ML Platform team
- Owner Supporting: Platform Engineering; Security Engineering
