# SAFE-K8S-0802-005 - GPU kernel module and driver binary integrity enforcement

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Advanced
- Classification: ai-specific
- Superseded By: SAFE-K8S-0802-008

## Statement

GPU kernel module loading must be restricted to signed and approved modules using Secure Boot and kernel module signing enforcement, and GPU driver binary paths must be monitored with file integrity tools to detect unauthorized replacement or tampering.

## Objective

Prevent unauthorized kernel module loading and driver binary replacement that could substitute legitimate GPU drivers with backdoored versions, compromising all workloads on the affected node.

## Rationale

Unauthorized kernel module loading can replace legitimate GPU drivers with backdoored versions that intercept GPU operations at the kernel level. Driver binary replacement on the filesystem achieves the same result by modifying the shared libraries that containers link against. These are OS-level and filesystem-level attacks with different enforcement points from firmware-level tampering (SAFE-K8S-0802-004), requiring distinct kernel and filesystem integrity controls.

## Scope

Applies to all Kubernetes GPU nodes running NVIDIA kernel modules (nvidia.ko, nvidia-uvm.ko) and GPU driver shared libraries. Covers kernel module signing enforcement, Secure Boot validation of GPU modules in the boot chain, and file integrity monitoring of GPU driver binary paths.

## Enforcement

- Primary Enforcement Point: Kernel module signing enforcement (CONFIG_MODULE_SIG_FORCE), Secure Boot configuration for GPU node boot chain, and file integrity monitoring tools on driver binary paths.
- Implementation Surface: Kernel module signing enforcement (CONFIG_MODULE_SIG_FORCE, modinfo nvidia signature validation), Secure Boot configuration for GPU node boot chain, file integrity monitoring tools (AIDE, OSSEC) on GPU driver binary paths (/usr/lib/x86_64-linux-gnu/libnvidia-*.so*), and runtime security tools detecting unauthorized module load events.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; GPU Infrastructure Engineering
