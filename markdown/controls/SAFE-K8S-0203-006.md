# SAFE-K8S-0203-006 - Cloud instance metadata service hardening and IMDSv2 configuration

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes nodes running on cloud infrastructure must enforce provider-specific instance metadata service hardening, including IMDSv2 or equivalent authenticated metadata access requirements with restrictive hop-limit or endpoint settings appropriate to the platform.

## Objective

Harden cloud instance metadata services at the node and provider configuration layer so instance metadata cannot be retrieved through insecure metadata service modes or permissive platform defaults.

## Rationale

Cloud provider metadata services expose instance identity and credentials. If provider metadata access remains in insecure modes such as IMDSv1, simple HTTP requests or SSRF paths can retrieve cloud credentials. Provider-side hardening reduces this attack surface before workload-level network controls are considered.

## Scope

Applies to Kubernetes nodes on AWS, GCP, Azure, and other cloud providers exposing instance metadata services. Covers node group, VM, or managed service metadata configuration that customers can set or must verify through provider documentation and platform APIs.

## Enforcement

- Primary Enforcement Point: Cloud infrastructure provisioning and managed service configuration governing instance metadata service behavior.
- Implementation Surface: Infrastructure-as-code templates, node group or VM metadata service settings, IMDSv2 or equivalent provider controls, hop-limit or endpoint restrictions, and managed service verification of metadata protection defaults.
- Owner Primary: Cloud Infrastructure
- Owner Supporting: Platform Engineering; Security Engineering
