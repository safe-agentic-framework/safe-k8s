# SAFE-K8S-0909-001 - Vector database access controls and encryption

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.9 - RAG Infrastructure Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Vector databases used for RAG retrieval (Milvus, Weaviate, Qdrant, pgvector, Chroma) must enforce authentication and RBAC for all client connections, must encrypt data at rest and in transit, and must implement collection-level access controls that restrict which applications and principals can read from or write to each vector collection.

## Objective

Prevent unauthorized access to vector embeddings that encode organizational knowledge, and prevent unauthorized modification of vector indexes that could enable context poisoning attacks.

## Rationale

RAG vector databases store document embeddings that encode the semantic content of an organization's knowledge base. Unauthorized read access enables embedding inversion attacks that reconstruct document content. Unauthorized write access enables index poisoning that controls what context reaches the LLM. Default installations of many vector databases ship with authentication disabled, creating an immediate exposure when deployed on shared Kubernetes clusters.

## Scope

Applies to all vector database instances used for RAG retrieval on Kubernetes. Covers authentication configuration, RBAC at collection level, TLS for client connections, PV encryption for storage volumes, and network policy isolation. Excludes general-purpose relational databases not used for vector storage.

## Enforcement

- Primary Enforcement Point: Vector database authentication and RBAC configuration, Kubernetes NetworkPolicy for vector database pods, and PV encryption configuration for vector storage volumes.
- Implementation Surface: Vector database platforms (Milvus, Weaviate, Qdrant, pgvector, Chroma) with authentication enabled and collection-level RBAC, Kubernetes ServiceAccount tokens or mTLS for service-to-service authentication, TLS configuration for all vector database client connections, PV encryption (SAFE-K8S-0701-003) for vector storage volumes, Kubernetes NetworkPolicy restricting access to retrieval service and ingestion pipeline pods only, and namespace or partition isolation for multi-tenant deployments.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Infrastructure Engineering
