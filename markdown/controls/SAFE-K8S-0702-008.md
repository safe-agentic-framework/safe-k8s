# SAFE-K8S-0702-008 - Tenant boundary enforcement through admission control, network policy, and RBAC

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must enforce tenant boundaries at the namespace level through the combined application of admission control policies preventing cross-tenant resource access, network policies implementing default-deny inter-namespace traffic with explicit allow rules, and RBAC bindings restricting each tenant's service accounts and users to their assigned namespaces.

## Objective

Create defense-in-depth tenant isolation by layering admission control, network policy, and RBAC to prevent any single control failure from compromising tenant boundaries.

## Rationale

No single Kubernetes isolation mechanism is sufficient on its own. RBAC prevents unauthorized API access but does not restrict network traffic. Network policies restrict traffic but do not prevent a user from referencing cross-namespace resources through the API. Admission control can enforce resource-level policies but does not control runtime network behavior. Only the combination of all three provides effective tenant isolation. A gap in any one layer can allow cross-tenant data access, service impersonation, or resource manipulation.

## Scope

Applies to all multi-tenant Kubernetes clusters. Each tenant namespace must have admission control policies, network policies, and RBAC bindings configured as a coordinated set. Exceptions for cross-namespace communication (e.g., shared services, monitoring) must be explicitly documented and reviewed. Applies to both AI and non-AI namespaces in shared clusters.

## Enforcement

- Primary Enforcement Point: Admission controllers (OPA Gatekeeper, Kyverno) for resource-level policies, NetworkPolicy objects for traffic isolation, and RoleBinding/ClusterRoleBinding objects for API access scoping, audited as a coordinated policy set.
- Implementation Surface: RBAC RoleBindings and ClusterRoleBindings audit for cross-namespace grant detection, default-deny ingress and egress NetworkPolicy objects per tenant namespace, Pod Security Admission (PSA) enforce mode configuration, admission control policies (Kyverno, OPA/Gatekeeper) for image allowlisting and required label validation, automated tenant boundary testing (cross-namespace pod exec, service access, PVC mount attempts), and cluster upgrade validation test suites.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Network Engineering
