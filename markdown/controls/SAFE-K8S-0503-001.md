# SAFE-K8S-0503-001 - Ingress controller hardening

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

Ingress controllers must enforce TLS termination with organization-approved cipher suites and minimum TLS version, integrate with a web application firewall for request filtering, and apply rate limiting to protect backend services from abuse and denial-of-service attacks.

## Objective

Harden the ingress path to ensure encrypted, filtered, and rate-limited traffic reaches backend services.

## Rationale

Ingress controllers are the primary entry point for external traffic into the cluster. Weak TLS configuration exposes traffic to interception and downgrade attacks. Without WAF integration, injection attacks, malformed requests, and known exploit payloads reach backend services directly. The absence of rate limiting allows volumetric abuse, credential stuffing, and application-layer denial-of-service attacks to overwhelm services.

## Scope

Applies to all ingress controllers (NGINX Ingress, Envoy, Traefik, cloud provider ALB/NLB controllers) in production clusters. TLS requirements apply to all externally facing endpoints. WAF and rate limiting requirements apply to internet-facing services and may be applied to internal services based on risk assessment.

## Enforcement

- Primary Enforcement Point: Ingress controller configuration, TLS secret management, WAF policy rules, and rate limiting annotations or custom resources.
- Implementation Surface: Ingress controller configurations (NGINX Ingress, Envoy, Traefik, cloud ALB/NLB controllers) with TLS termination settings (ssl-protocols TLSv1.3, approved cipher suites), WAF integration (ModSecurity embedded in NGINX, Coraza, AWS WAF, GCP Cloud Armor, Azure WAF), rate limiting annotations (limit-rps, limit-connections), request size limits (proxy-body-size), response buffer limits (proxy-buffer-size), request timeout configuration (proxy-read-timeout), TLS scanning tools (ssllabs, testssl.sh), and cert-manager for ingress TLS certificate management.
- Owner Primary: Network Engineering
- Owner Supporting: Application Security; Platform Engineering
