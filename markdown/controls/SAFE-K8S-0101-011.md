# SAFE-K8S-0101-011 - API server profiling disablement and approved admission controller configuration

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must disable unnecessary API server profiling or debug exposure and must restrict the admission controller chain to approved components.

## Objective

Reduce control-plane attack surface and policy bypass risk by hardening API server debug surfaces and admission controller behavior.

## Rationale

Exposed profiling and debug interfaces can leak sensitive runtime information and be abused for denial-of-service or reconnaissance. Unapproved admission controllers can weaken or bypass policy enforcement at the API server entry point.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that host sensitive data, shared workloads, or provide a path to production. For managed Kubernetes services, this control applies to all customer-exposed API server settings and to compensating controls where provider-managed settings are not directly configurable.

## Enforcement

- Primary Enforcement Point: API Server Surface Reduction and Admission Configuration
- Implementation Surface: Cluster provisioning templates, API server configuration management, managed Kubernetes security settings, and periodic configuration assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
