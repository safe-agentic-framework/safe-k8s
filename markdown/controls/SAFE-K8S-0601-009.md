# SAFE-K8S-0601-009 - Registry and runtime container vulnerability monitoring

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Container images must be scanned again at registry push time and monitored for vulnerability exposure at runtime, with continuous tracking of AI-specific dependency findings in deployed workloads and documented risk acceptance for unresolved critical or high-severity exposures.

## Objective

Maintain current visibility into image risk after build completion by rescanning pushed images and monitoring running workloads for newly disclosed or persistent vulnerabilities.

## Rationale

Build-time scanning alone becomes stale as new CVEs are disclosed and images age in registries or running clusters. Registry and runtime monitoring provide the continuous exposure picture needed to identify newly affected workloads, prioritize rebuilds, and track exceptions until remediation is complete.

## Scope

Applies to container registries and deployed workloads across all Kubernetes clusters, including runtime scanning of AI/ML images and continuous monitoring of native library exposure.

## Enforcement

- Primary Enforcement Point: Registry / Runtime Monitoring
- Implementation Surface: Registry webhook scanning, runtime scanning agents, exposure dashboards, and exception governance for unresolved findings.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
