# SAFE-K8S-0101-001 - API server hardening flags

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must disable anonymous API server authentication, disable unnecessary profiling/debug exposure, restrict the admission controller chain to approved components, and must not enable AlwaysAllow authorization. Audit logging configuration is addressed separately in SAFE-K8S-0101-006.

## Objective

Reduce control-plane attack surface by enforcing secure Kubernetes API server settings for authentication, debug surface reduction, and admission behavior.

## Rationale

The Kubernetes API server is the primary control-plane entry point. Misconfigured authentication, exposed profiling or debugging functionality, or weak admission configuration increase the risk of unauthorized access and policy bypass.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that host sensitive data, shared workloads, or provide a path to production. For managed Kubernetes services, this control applies to all customer-exposed API server settings and to compensating controls where provider-managed settings are not directly configurable.

## Enforcement

- Primary Enforcement Point: API Server Configuration
- Implementation Surface: Cluster provisioning templates, control-plane configuration management, managed Kubernetes security settings, and periodic configuration assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
