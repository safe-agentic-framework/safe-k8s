# SAFE-K8S-0403-013 - Credential scope drift monitoring and lifecycle metrics governance

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations running AI workloads at scale must monitor credential scope drift and publish credential lifecycle metrics, including issuance rate, average TTL, orphan rate, and scope exceptions, to govern credential sprawl across the platform.

## Objective

Provide governance over credential breadth and lifecycle behavior by measuring how credentials are scoped, used, and accumulated across AI workloads.

## Rationale

Credential sprawl is not only a count problem but also a scope and lifecycle problem. Credentials broaden over time through exceptions, and platforms lose visibility into issuance and orphan trends unless metrics are published. This control isolates scope-drift monitoring and lifecycle metrics so teams can independently prove that credential sprawl is being governed and trended.

## Scope

Applies to all credentials issued to AI and ML workloads including Vault dynamic secrets, cloud IAM temporary credentials, Kubernetes service account tokens, API keys for external services, and model registry authentication tokens. Covers scope drift analysis and platform-level credential lifecycle metrics.

## Enforcement

- Primary Enforcement Point: Credential scope drift detection and lifecycle governance telemetry.
- Implementation Surface: Scope drift analysis comparing granted versus used permissions, credential governance dashboards, and metric publication workflows for issuance rate, TTL, orphan rate, and scope exceptions.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering; Cloud Platform Engineering
