# SAFE-K8S-0104-001 - CIS Kubernetes Benchmark automated assessment and remediation

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must be assessed against the CIS Kubernetes Benchmark using automated tooling (kube-bench or equivalent) on a recurring schedule, with failing checks tracked through a remediation workflow and any accepted exceptions formally documented with risk owner approval and periodic re-review.

## Objective

Establish a measurable, repeatable security baseline for Kubernetes cluster configuration by automating CIS Benchmark assessment, remediating findings, and governing exceptions.

## Rationale

Manual configuration review does not scale and is prone to drift. The CIS Kubernetes Benchmark provides a consensus security baseline covering API server, etcd, controller-manager, scheduler, kubelet, and policy configuration. Without automated assessment, configuration drift introduces security gaps that go undetected until an incident. Undocumented exceptions create hidden risk accepted without accountability.

## Scope

Applies to all production clusters. Covers CIS Benchmark sections for control-plane components, worker nodes, and policies. For managed Kubernetes services, applies to the CIS managed benchmark profile (customer-configurable checks) and to documenting provider-managed checks that cannot be assessed directly.

## Enforcement

- Primary Enforcement Point: Audit / Telemetry
- Implementation Surface: Automated CIS Benchmark scanning pipeline (CI/CD or scheduled job), remediation tracking system (issue tracker or GRC platform), and exception approval workflow.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Governance Risk and Compliance
