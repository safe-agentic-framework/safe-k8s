# SAFE-K8S-0905-023 - OCI model artifact digest pinning and tag immutability

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Models stored and promoted through OCI registries must use digest-pinned references and registry tag immutability controls so that deployment and promotion workflows cannot silently resolve mutable tags to different model artifacts.

## Objective

Ensure that model promotion and deployment workflows consume immutable OCI artifact references rather than mutable tags that can be retargeted without review.

## Rationale

Even when artifacts are signed, mutable tags and non-pinned references allow a registry operator or compromised pipeline to redirect a deployment to a different model than the one that was reviewed. Digest pinning and tag immutability prevent silent artifact substitution in model registries and deployment manifests.

## Scope

Applies to OCI registries, model registries backed by OCI artifacts, deployment manifests, and promotion workflows that reference trained models. Covers digest pinning, immutable tags, and registry configuration that blocks tag reuse or overwrite.

## Enforcement

- Primary Enforcement Point: OCI registry configuration and deployment manifests that reference model artifacts.
- Implementation Surface: OCI registries such as Harbor, ECR, GCR, or ACR configured for immutable tags, deployment and promotion manifests referencing models by digest (@sha256:...), policy checks that reject mutable-tag model references, and audit records showing registry immutability enforcement.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; DevOps Engineering
