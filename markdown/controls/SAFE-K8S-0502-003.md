# SAFE-K8S-0502-003 - IPAM security and IP spoofing protection

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.2 - CNI Plugins and Pod Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

IP address management must be configured to prevent IP address exhaustion through adequate CIDR allocation and monitoring, and IP spoofing protection must be enabled to ensure pods cannot forge source IP addresses to bypass network policies or impersonate other workloads.

## Objective

Maintain IP address availability for workload scheduling and prevent network-layer identity spoofing that could circumvent policy enforcement.

## Rationale

IP exhaustion in pod CIDR ranges causes scheduling failures and service outages, which can be exploited as a denial-of-service vector. IP spoofing allows a malicious pod to impersonate another workload's IP address, bypassing NetworkPolicy rules that rely on source IP for access decisions. Without spoofing protection, the entire network policy model can be subverted by a single compromised pod.

## Scope

Applies to all Kubernetes clusters. IPAM monitoring must cover pod CIDR, service CIDR, and node CIDR ranges. IP spoofing protection applies to all CNI configurations and must be verified on both managed and self-managed clusters.

## Enforcement

- Primary Enforcement Point: CNI plugin IPAM configuration and anti-spoofing enforcement at the pod virtual network interface (veth pair or eBPF datapath).
- Implementation Surface: CNI plugin IPAM configuration (pod CIDR allocation, service CIDR, node CIDR), IPAM utilization monitoring and alerting (80% threshold), CNI anti-spoofing enforcement at veth pair or eBPF datapath (Cilium and Calico default anti-spoofing), and IP spoofing test validation confirming forged source IP packets are dropped.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Operations
