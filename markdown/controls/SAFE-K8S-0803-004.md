# SAFE-K8S-0803-004 - DPU and SmartNIC firmware integrity and trust boundaries

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.3 - High-Performance Interconnect Security
- Maturity: Advanced
- Classification: ai-specific
- Superseded By: SAFE-K8S-0803-005

## Statement

DPU (Data Processing Unit) and SmartNIC devices used for AI workload acceleration or network offload must maintain firmware integrity through cryptographic verification, enforce clear trust boundaries between offloaded and host-processed functions, and restrict policy enforcement capabilities to authorized administrators.

## Objective

Ensure that DPUs and SmartNICs operating as independent compute elements with their own firmware and policy engines do not become a vector for bypassing host-level security controls or injecting unauthorized network policies into AI workload traffic paths.

## Rationale

DPUs run independent operating systems and firmware with direct access to network traffic and storage I/O, operating below the host OS trust boundary. Compromised DPU firmware can intercept, modify, or redirect all network traffic including distributed training data, bypass host-level firewalls and network policies, and manipulate storage I/O to tamper with model checkpoints. The offload trust boundary between host and DPU creates a security gap where policies enforced at the DPU level may not be visible to host-level security monitoring.

## Scope

Applies to Kubernetes nodes equipped with DPUs (NVIDIA BlueField, AMD/Pensando, Intel IPU) or SmartNICs with programmable firmware. Covers firmware integrity, DPU-enforced network policies, and offload trust boundary configuration. Standard NICs without programmable firmware or policy engines are not in scope.

## Enforcement

- Primary Enforcement Point: DPU firmware management platform (BlueField BMC, DPU OS), Secure Boot configuration for DPU firmware chain, DPU policy engine access controls, and host-to-DPU trust boundary configuration.
- Implementation Surface: DPU firmware management platforms (NVIDIA BlueField BMC, AMD Pensando, Intel IPU management interfaces), DPU Secure Boot configuration and firmware cryptographic signature verification, DPU ARM core OS and policy engine access controls, host-to-DPU trust boundary documentation and configuration, DPU-enforced NetworkPolicy consistency verification against Kubernetes-level policies, SmartNIC programmable firmware integrity monitoring, and DPU management interface RBAC restricting tenant pod access.
- Owner Primary: Network Engineering
- Owner Supporting: GPU Infrastructure Engineering; Security Engineering; Platform Engineering
