# SAFE-K8S-0104-006 - Kubernetes CVE monitoring and prioritization

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must establish active subscriptions to Kubernetes security announcement channels (kubernetes-security-announce mailing list, CVE databases, and provider advisories) and maintain a documented CVE prioritization framework that classifies vulnerabilities by CVSS score, exploitability, and cluster exposure to drive timely triage decisions.

## Objective

Ensure security teams receive timely notification of Kubernetes vulnerabilities and apply a consistent prioritization framework that focuses remediation effort on the highest-risk exposures first.

## Rationale

Kubernetes CVEs are publicly disclosed with increasing frequency. Without active monitoring, security teams may be unaware of critical vulnerabilities affecting their clusters. Without a prioritization framework, all CVEs receive equal treatment, delaying response to the most critical issues. Structured triage is a prerequisite for any effective patch response.

## Scope

Applies to all production and non-production clusters. Covers Kubernetes core component CVEs, container runtime CVEs, and critical add-on vulnerabilities (CNI, CSI, ingress controllers). For managed Kubernetes services, applies to monitoring provider patch timelines and coordinating shared responsibility for component-level CVEs.

## Enforcement

- Primary Enforcement Point: Governance / Process
- Implementation Surface: Security announcement monitoring integration (mailing list subscriptions, RSS feeds, vulnerability scanner feeds) and CVE prioritization workflow documentation.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Incident Response
