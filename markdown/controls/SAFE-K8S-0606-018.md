# SAFE-K8S-0606-018 - Zero-trust CI/CD handoff verification and independent evidence generation

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CI/CD pipelines must verify trust at every stage transition through authenticated handoffs, artifact integrity verification, and independently generated evidence artifacts whose trust does not depend solely on the build process, consistent with NIST SP 800-204D zero-trust CI/CD principles.

## Objective

Ensure stage-to-stage trust in CI/CD is explicitly verified and supported by evidence sources that remain trustworthy even if the build process itself is compromised.

## Rationale

A compromised CI/CD system can falsify success if downstream stages trust pipeline assertions without independent verification. Zero-trust CI/CD requires authenticated handoffs and independently trusted evidence so compromise in one stage does not automatically invalidate assurance for the rest of the delivery chain.

## Scope

Applies to all CI/CD pipelines that transfer artifacts, attestations, or deployment intent between stages prior to Kubernetes deployment. Covers authenticated stage transitions, integrity checks, and independent evidence generation components.

## Enforcement

- Primary Enforcement Point: Inter-stage trust verification, authenticated stage identities, and evidence generation components.
- Implementation Surface: Artifact digest verification between stages, authenticated handoff mechanisms such as OIDC or workload identities, independently generated signatures and attestations, and pipeline trust models showing evidence generation components are logically separate from and more trusted than the build step they validate.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Platform Engineering
