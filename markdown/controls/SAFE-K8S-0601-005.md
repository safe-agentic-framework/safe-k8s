# SAFE-K8S-0601-005 - Container image and artifact retention policy

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must define and enforce retention policies for container images, Helm charts, ML model artifacts, and associated integrity verification data (signatures, attestations, SBOMs) stored in registries. Retention policies must specify minimum retention periods for each artifact type, protect artifacts referenced by running deployments from garbage collection, and ensure that integrity metadata is retained for at least as long as the artifact it describes. Policies must be codified in registry configuration or enforced through automated lifecycle management tooling, not solely documented in runbooks.

## Objective

Ensure that software release artifacts and their integrity metadata are retained long enough to support vulnerability investigation, incident response, audit, and rollback while preventing unbounded storage growth.

## Rationale

Without explicit retention policies, organizations risk losing the provenance and integrity data needed to investigate vulnerabilities in previously deployed software, or conversely accumulating unbounded artifact storage with no lifecycle governance. SSDF task PS.3.1 requires secure archival of release artifacts.

## Scope

All container registries, Helm chart repositories, ML model registries, and OCI artifact stores used by the organization.

## Enforcement

- Primary Enforcement Point: registry
- Implementation Surface: Registry garbage collection policies, tag immutability settings, retention rules in Harbor/ECR/GCR/ACR, OCI artifact lifecycle policies, Helm chart museum or ChartMuseum retention configuration, ML model registry version retention settings.
- Owner Primary: Platform Engineering
- Owner Supporting: DevSecOps / Release Engineering; ML Platform Engineering
