# SAFE-K8S-0201-004 - Kubelet configuration file permissions and node-level audit controls

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.1 - Kubelet and Node Configuration Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubelet configuration files, kubeconfig files, and certificate key material on each node must be owned by root with permissions no more permissive than 0644 for configuration files and 0600 for private keys, and the kubelet systemd unit must be configured with appropriate security directives to limit process capabilities.

## Objective

Prevent unauthorized local users or compromised workloads from reading or modifying kubelet configuration, credentials, or certificate material on the node filesystem.

## Rationale

Kubelet configuration files contain authentication credentials, certificate paths, and runtime parameters that control node behavior. Overly permissive file permissions allow any local process or container escape to read kubelet credentials and impersonate the node to the API server. Insecure systemd unit configuration can allow the kubelet process to be manipulated or restarted with altered parameters, bypassing hardening controls.

## Scope

Applies to all files in /etc/kubernetes/, /var/lib/kubelet/, and kubelet certificate directories on every cluster node. Managed Kubernetes services where the node filesystem is provider-controlled satisfy this control through provider hardening attestation. Nodes using read-only or immutable root filesystems may document this as a compensating control.

## Enforcement

- Primary Enforcement Point: Node image build pipeline, configuration management tooling, and periodic file permission audit scans.
- Implementation Surface: Node image build pipeline, configuration management tooling (Ansible, Chef), systemd unit hardening directives (ProtectSystem, ProtectHome, NoNewPrivileges, PrivateTmp), file permission auditing, and CIS Benchmark scanning (kube-bench checks 4.1.x).
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; OS/Infrastructure
