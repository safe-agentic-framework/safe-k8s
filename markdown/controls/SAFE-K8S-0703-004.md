# SAFE-K8S-0703-004 - GPU resource abuse detection and prevention

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.3 - Resource Governance and Priority
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must implement detection and prevention controls for GPU resource squatting, quota circumvention, and unauthorized GPU usage including cryptomining, personal workloads, and resource policy violations, using GPU utilization monitoring, workload fingerprinting, and admission control enforcement to identify and terminate non-sanctioned GPU consumption.

## Objective

Prevent misuse of expensive GPU resources by detecting unauthorized workload patterns and enforcing acceptable use policies on GPU-equipped nodes.

## Rationale

GPU resources are high-value targets for abuse. Cryptomining malware specifically targets GPU-equipped nodes for maximum hash rates. Internal users may run personal ML experiments or reserve GPUs without active use (resource squatting) to guarantee future availability. Sophisticated users may circumvent quotas by splitting workloads across namespaces or manipulating resource requests. Without active detection, these abuses waste resources, increase costs, and in the case of cryptomining may indicate a broader security compromise.

## Scope

Applies to all GPU-equipped nodes in Kubernetes clusters. Monitoring must cover GPU utilization patterns, process identification on GPU devices, and quota consumption analytics. Detection must identify cryptomining signatures, prolonged low-utilization GPU reservations, and cross-namespace quota circumvention patterns. Enforcement must include automated response actions for confirmed abuse.

## Enforcement

- Primary Enforcement Point: GPU monitoring agents on GPU nodes, workload process inspection on GPU devices, admission control preventing quota circumvention patterns, and automated response controllers that terminate or quarantine abusive workloads.
- Implementation Surface: NVIDIA DCGM GPU telemetry and Prometheus metrics for workload fingerprinting and utilization profiling, admission control policies (Kyverno, OPA/Gatekeeper) blocking hostPath volume mounts to /dev/nvidia* device paths, nvidia.com/gpu device plugin resource request enforcement, GPU-aware runtime monitoring (Falco, Tetragon) for anomalous GPU kernel execution patterns, cryptomining detection signatures (sustained 100% utilization with minimal memory allocation changes), label enforcement from SAFE-K8S-0702-005 for workload allowlisting, and security team alerting for non-sanctioned GPU consumption.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering
