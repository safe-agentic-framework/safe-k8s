# SAFE-K8S-0905-007 - Model format security and deserialization controls

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0905-032

## Statement

Organizations must enforce safe model serialization formats for all model loading operations, must block or quarantine models using unsafe deserialization (Python pickle, torch.load without weights_only=True), must validate model file structure and metadata before loading, and must maintain an allowlist of approved model formats (SafeTensors, ONNX, GGUF) for production deployment.

## Objective

Prevent arbitrary code execution through malicious model files by enforcing safe serialization formats and validating model file integrity before any deserialization occurs in training or inference workloads.

## Rationale

Python pickle-based model formats (PyTorch .pt/.pth, joblib, cloudpickle) execute arbitrary code during deserialization. An attacker who can substitute a model file with a pickle-based payload achieves code execution in the context of the loading process, typically with full access to GPU resources, training data, and inference infrastructure. SafeTensors and other safe formats store only tensor data without executable code, eliminating this attack vector. Without format enforcement, a single malicious model file can compromise an entire inference cluster.

## Scope

Applies to all model loading operations on Kubernetes, including inference servers, training pipelines loading pre-trained weights, and fine-tuning jobs. Covers model format validation, deserialization controls, format allowlisting, CI/CD static analysis for unsafe deserialization calls, and container image hardening to remove unsafe deserialization libraries. Excludes research and experimentation environments where controlled use of pickle may be acceptable with compensating controls.

## Enforcement

- Primary Enforcement Point: Model loading configuration in inference servers and training jobs, admission controllers validating model file formats, CI/CD pipeline gates that scan model files before registry publication, and model promotion gate format verification.
- Implementation Surface: Model format allowlist policy (SafeTensors, ONNX, GGUF), admission controllers (Kyverno, Gatekeeper) rejecting pods mounting pickle-based model files or using unsafe deserialization, inference server safe deserialization configuration (torch.load with weights_only=True), CI/CD static analysis flagging unsafe torch.load calls, model file structure validation (header magic bytes, size bounds, ONNX operator allowlist, TensorFlow custom op validation), pickle opcode analysis tools (fickling), seccomp and AppArmor profiles restricting deserialization syscalls, and SafeTensors migration tooling and tracking.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
