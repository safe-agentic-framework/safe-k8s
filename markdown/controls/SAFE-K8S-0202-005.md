# SAFE-K8S-0202-005 - Container escape prevention through kernel security profiles

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Nodes must implement runtime-level protections including default seccomp profiles for all containers, AppArmor or SELinux mandatory access control policies, and user namespace isolation where supported, to reduce the exploitability of runtime vulnerabilities that could enable container escape.

## Objective

Reduce the exploitability of kernel and runtime vulnerabilities by enforcing defense-in-depth kernel security controls that constrain container capabilities even when other isolation layers are bypassed.

## Rationale

Kernel and runtime vulnerabilities can be exploited for container escape even without direct runtime socket access. Seccomp filtering restricts the syscall surface available to containers, blocking exploitation of many kernel vulnerabilities. Mandatory access control (AppArmor/SELinux) confines container processes to declared access patterns. User namespace isolation prevents UID 0 inside a container from mapping to host root. Without these layered defenses, a single runtime CVE can yield full host compromise.

## Scope

Applies to all production cluster nodes. Default seccomp profiles must be applied to all workloads unless a documented exception exists. AppArmor or SELinux must be active on the host and applied to container workloads. User namespace isolation should be enabled where the container runtime and kernel version support it.

## Enforcement

- Primary Enforcement Point: Node-level kernel security configuration (seccomp default profiles, AppArmor/SELinux policy loading) and Pod Security Admission enforcing security context requirements.
- Implementation Surface: Default seccomp profile configuration (RuntimeDefault or custom), AppArmor profile loading and annotation enforcement, SELinux context assignment, user namespace remapping configuration, Pod Security Admission or equivalent enforcing securityContext constraints, and runtime detection tools for syscall monitoring.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; OS/Infrastructure
