# SAFE-K8S-0102-017 - etcd backup repository access audit logging and review

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must generate auditable records for all access to etcd backup repositories and restore media, and those records must be reviewed or alertable for unauthorized or anomalous access activity.

## Objective

Ensure that access to etcd backup repositories is observable and reviewable so unauthorized reads, writes, or restore-media handling can be detected and investigated.

## Rationale

Least-privilege repository policies are insufficient if access activity is not observable. Auditable records and review workflows provide the evidence needed to detect misuse, investigate suspicious access, and demonstrate separation-of-duties over backup repositories.

## Scope

Applies to all production clusters. Covers object storage access logging, repository audit trails, restore-media access records, and review or alert workflows for etcd backup repositories.

## Enforcement

- Primary Enforcement Point: Backup Repository Access Monitoring
- Implementation Surface: Backup repository access logs, object storage audit events, restore-media access records, and periodic review or alert workflows for monitored repository access.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Backup and Recovery Operations
