# SAFE-K8S-0903-002 - Autoscaling safety and queue security

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.3 - Inference Resilience, Adversarial Defense, and Resource Controls
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0903-005

## Statement

GPU-aware Horizontal Pod Autoscaler configurations for inference workloads must enforce maximum replica bounds and cooldown periods to prevent denial-of-wallet attacks, and inference request queues must implement priority controls, timeout enforcement, and queue depth limits to prevent backpressure attacks.

## Objective

Prevent cost-based denial-of-service through uncontrolled GPU autoscaling and protect inference availability through hardened request queue management.

## Rationale

GPU-based inference pods are extremely expensive to scale. Without maximum replica bounds and cooldown periods, an attacker can trigger runaway autoscaling by flooding the inference endpoint, causing massive cloud costs (denial-of-wallet). Without queue depth limits and timeouts, an attacker can fill the request queue with slow or malformed requests, creating backpressure that blocks legitimate inference traffic.

## Scope

Applies to all autoscaled inference deployments on Kubernetes using HPA, KEDA, or custom GPU-aware autoscalers. Covers HPA configuration, replica bounds, cooldown settings, and inference request queue configurations. Excludes training job scheduling, which is covered by SAFE-K8S-0901-006.

## Enforcement

- Primary Enforcement Point: HPA resource definitions, KEDA ScaledObject configurations, inference server queue settings, and API gateway rate limiting policies.
- Implementation Surface: Kubernetes HorizontalPodAutoscaler (autoscaling/v2) with maxReplicas bounds, stabilizationWindowSeconds cooldown, and rate-limited scale-up policies, KEDA ScaledObject configurations, custom metrics adapters (Prometheus Adapter) for GPU utilization and inference queue depth, API gateway rate limiting returning HTTP 429, inference request queue depth limits and per-request timeout settings, priority queuing configuration, and FinOps budget integration triggering scale-up disablement.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Platform Engineering; FinOps
