# SAFE-K8S-0102-010 - etcd certificate rotation execution and testing

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must implement automated rotation of etcd server certificates, peer certificates, and API server-to-etcd client certificates before expiration, and must test and document the rotation process to verify that certificate replacement completes without control-plane disruption.

## Objective

Ensure that all etcd-related certificates are rotated automatically and safely so expired credentials do not cause loss of etcd availability or broken control-plane trust paths.

## Rationale

Certificate lifecycle failures often occur in the execution path: rotation jobs are missing, one certificate class is omitted, or the process is never rehearsed. This control isolates the operational rotation and testing obligation from certificate policy and monitoring so teams can verify end-to-end replacement behavior independently.

## Scope

Applies to all self-managed Kubernetes clusters using etcd. Covers etcd server, peer, and API server client certificates and the tooling or automation responsible for rotating them.

## Enforcement

- Primary Enforcement Point: Certificate Rotation Workflow
- Implementation Surface: kubeadm or PKI automation configuration, cert-manager or equivalent issuance workflows, etcd manifest certificate references, and tested rotation runbooks.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
