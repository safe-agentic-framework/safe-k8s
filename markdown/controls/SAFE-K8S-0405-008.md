# SAFE-K8S-0405-008 - Break-glass emergency access procedures

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must define and test break-glass emergency access procedures that enable authorized personnel to regain cluster access when normal authentication pathways (OIDC providers, external identity systems) are unavailable, with all break-glass access fully audited and automatically time-limited.

## Objective

Ensure that legitimate administrators can access Kubernetes clusters during authentication infrastructure outages or security incidents without permanently weakening the authentication posture.

## Rationale

OIDC providers and external identity systems can become unavailable during outages or targeted attacks. Without pre-defined break-glass procedures, administrators may resort to ad hoc workarounds (shared kubeconfigs, disabling authentication) that create persistent security gaps. Formal break-glass procedures with audit trails and automatic expiration provide emergency access while maintaining accountability.

## Scope

Applies to all production Kubernetes clusters. Covers break-glass credential provisioning, storage, access procedures, audit logging, and automatic expiration. Break-glass procedures must be tested periodically to verify they function when needed.

## Enforcement

- Primary Enforcement Point: Break-glass credential storage (sealed envelope, hardware security module, or emergency credential vault), audit logging of all break-glass access, and automatic credential expiration enforcement.
- Implementation Surface: Emergency kubeconfig or client certificate generation with short TTL, sealed envelope or HSM-based credential storage, Kubernetes audit log configuration capturing break-glass access, automatic credential expiration or revocation after emergency window, and documented runbooks with periodic testing schedules.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Identity and Access Management
