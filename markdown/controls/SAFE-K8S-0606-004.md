# SAFE-K8S-0606-004 - GitOps repository security

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

GitOps deployment repositories must enforce repository access controls with least-privilege service accounts, require signed commits on deployment branches, enforce branch protection rules including PR approval and status checks, and preserve release metadata for all deployed packages, versions, and configuration files.

## Objective

Ensure the integrity of the Git-based source of truth for deployments by enforcing access controls, commit signing, and branch protection on deployment repositories.

## Rationale

Without repository-level controls, adversaries with repository access can inject malicious manifests through unsigned commits or force-pushed changes that bypass code review. Least-privilege service accounts limit the blast radius of credential compromise. Branch protection ensures all changes undergo review and pass status checks before merging to deployment branches. Release metadata preservation enables accurate reconstruction of what was deployed and when during incident response.

## Scope

Applies to all Git repositories used as deployment sources for Flux or ArgoCD in production and staging environments. Covers repository access controls, commit signing enforcement, and branch protection configuration. Non-production experimentation repositories may have relaxed controls with documented justification.

## Enforcement

- Primary Enforcement Point: Git repository platform settings (GitHub, GitLab, Bitbucket).
- Implementation Surface: Flux GitRepository source verification (verify option for signed commits) and ArgoCD GPG commit signature verification, Git repository branch protection rules (PR approval, status checks, no force-push on main/release branches), Flux/ArgoCD service account RBAC with least-privilege read-only access to deployment repositories, and release metadata preservation for deployed package versions and configuration files.
- Owner Primary: Platform Engineering
- Owner Supporting: DevSecOps / Release Engineering; Security Engineering
