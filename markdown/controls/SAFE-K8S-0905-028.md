# SAFE-K8S-0905-028 - ML framework and Python dependency vulnerability management

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must perform software composition analysis and vulnerability triage for ML frameworks and Python ecosystem dependencies used in training and inference environments, and must track remediation or compensating actions for identified vulnerabilities.

## Objective

Reduce exploitability of vulnerable ML frameworks and Python dependencies by continuously identifying, prioritizing, and remediating defects in those software components.

## Rationale

ML workloads depend on large dependency graphs across frameworks and Python packages. Without focused vulnerability management, known defects in PyTorch, TensorFlow, JAX, model-loading libraries, and transitive Python dependencies remain deployable and exploitable across training and inference environments.

## Scope

Applies to ML software dependencies used in training and inference environments on Kubernetes. Covers ML frameworks, model-loading libraries, Python packages, and transitive dependencies. Excludes CUDA and GPU accelerator library vulnerability management, which is covered by SAFE-K8S-0905-029.

## Enforcement

- Primary Enforcement Point: CI/CD and image-build SCA scanning plus framework dependency triage workflow.
- Implementation Surface: SCA scanning for PyTorch, TensorFlow, JAX, Python packages, model-loading libraries, and related transitive dependencies; vulnerability triage workflows; dependency update records; compensating-control approvals; and tracking of deserialization or dependency-related exposure in ML libraries.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; DevOps Engineering
