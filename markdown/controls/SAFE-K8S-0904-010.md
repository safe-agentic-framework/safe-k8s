# SAFE-K8S-0904-010 - Pipeline definition signing, immutability, and rollback integrity

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0904-017

## Statement

Pipeline definitions must be protected through signed manifests, immutable version storage, and controlled rollback procedures to prevent unauthorized modification or execution of tampered ML workflows.

## Objective

Ensure that pipeline definitions cannot be silently modified and that only authenticated, versioned, and recoverable pipeline specifications are executed.

## Rationale

Pipeline definitions govern the full ML workflow. If manifests can be modified without signing, immutability, and rollback discipline, an attacker can inject malicious stages, exfiltrate data, or deploy compromised models while obscuring provenance and recovery paths.

## Scope

Applies to ML pipeline definitions, templates, and workflow manifests stored and executed on Kubernetes-based orchestration platforms such as Argo Workflows and Kubeflow Pipelines. Covers definition signing, immutable version storage, signature verification workflows, and controlled rollback of pipeline definitions. Excludes experiment metadata access control, which is covered by SAFE-K8S-0904-009.

## Enforcement

- Primary Enforcement Point: Pipeline definition repository, signature verification point, and rollback process.
- Implementation Surface: Signed pipeline manifests, immutable workflow or template version storage, signature verification before execution or admission, and rollback-capable version management for workflow definitions.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
