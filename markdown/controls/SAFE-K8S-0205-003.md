# SAFE-K8S-0205-003 - Node-level firewall and Kubernetes network policy integration

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.5 - kube-proxy and Node Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes nodes must implement host-level firewall rules that restrict inbound and outbound traffic to required cluster communication ports and protocols, and these rules must be validated for compatibility with the cluster CNI plugin's network policy enforcement to avoid conflicts or gaps between node firewall and pod network policy layers.

## Objective

Provide defense-in-depth network segmentation at the node level that complements Kubernetes network policies, ensuring that node-to-node and node-to-external traffic is restricted to authorized communication patterns.

## Rationale

Kubernetes network policies operate at the pod level and depend on the CNI plugin for enforcement. Node-level firewall rules provide an independent defense layer that restricts traffic even if the CNI plugin is misconfigured, bypassed, or compromised. Without node firewalls, compromised nodes can communicate freely with any network endpoint, enabling lateral movement to non-Kubernetes infrastructure, data exfiltration to external destinations, and access to management networks. However, improperly configured node firewalls can conflict with CNI overlay networks and break legitimate cluster communication.

## Scope

Applies to all production cluster nodes. Cloud environments should use security groups or VPC firewall rules as the node-level firewall layer. On-premises environments must configure iptables, nftables, or firewalld rules. Firewall rules must account for CNI-specific port requirements (e.g., VXLAN, Geneve, WireGuard overlay ports). Managed Kubernetes services must document provider-managed network controls.

## Enforcement

- Primary Enforcement Point: Cloud security group or VPC firewall rules applied via infrastructure-as-code templates, or host-level firewall configuration in node provisioning automation.
- Implementation Surface: Cloud security groups (AWS), VPC firewall rules (GCP), NSGs (Azure), on-premises firewall tools (nftables, iptables), node provisioning automation, firewall rule verification scripts, and CNI port allowance configuration (VXLAN, Geneve, WireGuard).
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Cloud Infrastructure; Security Engineering
