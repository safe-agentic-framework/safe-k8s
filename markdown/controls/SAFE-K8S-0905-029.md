# SAFE-K8S-0905-029 - CUDA and GPU accelerator dependency vulnerability management

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must perform vulnerability triage and remediation tracking for CUDA toolkits and GPU accelerator libraries used in training and inference environments.

## Objective

Reduce exploitability of vulnerable accelerator software components by continuously identifying, prioritizing, and remediating defects in CUDA and related GPU libraries.

## Rationale

CUDA, cuDNN, NCCL, and related accelerator libraries are high-privilege, performance-critical dependencies in ML workloads. If their vulnerabilities are not managed separately, exploitable defects can persist across GPU-backed training and inference environments.

## Scope

Applies to CUDA toolkits and GPU accelerator libraries used in Kubernetes-based training and inference environments. Covers CUDA, cuDNN, NCCL, and similar accelerator dependencies. Excludes framework and Python package vulnerability management, which is covered by SAFE-K8S-0905-028.

## Enforcement

- Primary Enforcement Point: GPU image and accelerator library dependency review workflow.
- Implementation Surface: Vulnerability scanning, advisory review, remediation tracking, and update records for CUDA toolkits, cuDNN, NCCL, and related accelerator libraries in training and inference images.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; DevOps Engineering
