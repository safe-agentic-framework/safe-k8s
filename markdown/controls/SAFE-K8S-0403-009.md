# SAFE-K8S-0403-009 - Automated secret leak detection and response

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must deploy automated secret leak detection tools to identify credential exposure in source repositories, container images, CI/CD artifacts, runtime logs, and log aggregation systems, and must implement incident response procedures for rapid revocation of detected leaks.

## Objective

Ensure rapid detection and response when secrets are inadvertently exposed in any artifact or log, minimizing the window of exploitation for leaked credentials.

## Rationale

Exposed secrets in repository commits, container image layers, CI/CD pipeline logs, or runtime output may go unnoticed for months without automated detection, providing persistent unauthorized access. Automated scanning across all artifact types combined with rapid revocation procedures closes this gap.

## Scope

Applies to all environments where secrets may be inadvertently exposed including source repositories, container image registries, CI/CD pipeline artifacts and logs, runtime log aggregation, and runtime environments. Covers detection tooling deployment, alerting, and incident response for leaked credentials.

## Enforcement

- Primary Enforcement Point: Automated secret scanning tools in CI/CD pipelines, pre-commit hooks, container image scanning, and log aggregation systems with incident response integration.
- Implementation Surface: Secret scanning tools (GitLeaks, TruffleHog, GitHub push protection) in CI/CD pipelines and pre-commit hooks, container image layer scanning for embedded credentials, log aggregation secret pattern filtering, runtime log monitoring for leaked credential patterns, alert routing to security operations, and incident response runbooks for credential revocation.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Application Development
