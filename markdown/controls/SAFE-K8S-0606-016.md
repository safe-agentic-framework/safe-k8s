# SAFE-K8S-0606-016 - Git-only production deployment path and direct-change prohibition

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production clusters managed by GitOps must prohibit direct runtime changes through cluster RBAC and enforce Git as the sole approved deployment path, with auditable break-glass logging for any emergency direct access.

## Objective

Prevent human or ad hoc tooling from bypassing GitOps by enforcing Git as the only approved production change path and tightly restricting direct cluster modification rights.

## Rationale

Even strong drift detection is insufficient if human operators can routinely modify production state outside Git. Restricting direct API changes and requiring auditable break-glass access preserves review integrity, deployment discipline, and accountability for exceptional interventions.

## Scope

Applies to production namespaces and clusters governed through GitOps. Covers RBAC restrictions on direct create, update, and delete operations and break-glass logging for emergency exceptions.

## Enforcement

- Primary Enforcement Point: Cluster RBAC and operational controls governing production change paths.
- Implementation Surface: Cluster RBAC denying create, update, and delete permissions on deployed resources for human users in production namespaces, GitOps-only deployment operating procedures, and audit logging for break-glass direct access with justification.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; SRE / Operations
