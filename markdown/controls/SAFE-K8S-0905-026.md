# SAFE-K8S-0905-026 - Approved external model source allowlist governance and review

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must maintain a curated allowlist of approved external model sources and must periodically review and update that allowlist.

## Objective

Restrict external model ingestion to a governed set of approved sources through documented allowlist curation and periodic review.

## Rationale

Without a curated and maintained allowlist, engineers can source pre-trained models and checkpoints from arbitrary public repositories, including typosquatted or impersonated publishers distributing malicious artifacts under trusted names.

## Scope

Applies to all external model sources accessed from Kubernetes infrastructure. Covers approved-source criteria, repository URL and publisher allowlists, and periodic governance review of permitted external model sources. Excludes enforcement through network and admission controls, which is covered by SAFE-K8S-0905-027.

## Enforcement

- Primary Enforcement Point: Approved-source policy and allowlist management process.
- Implementation Surface: Approved external model source allowlist covering repository URLs, organization identifiers, or registry namespaces, with documented review dates, update procedures, and governance ownership for deciding which external sources are permitted.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Governance
