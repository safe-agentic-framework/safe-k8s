# SAFE-K8S-0801-008 - MPS and time-slicing memory isolation verification and side-channel assessment

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Advanced
- Classification: ai-specific
- Superseded By: SAFE-K8S-0801-013

## Statement

Clusters using NVIDIA MPS or GPU time-slicing must verify GPU memory isolation, prevent cross-tenant data remnants between workload transitions, and perform side-channel resistance assessments for shared GPU deployments.

## Objective

Provide technical assurance that weaker GPU-sharing modes do not expose recoverable memory remnants or unassessed side-channel leakage risks between co-located workloads.

## Rationale

Even when use of MPS or time-slicing is formally approved, organizations still need independent evidence that memory remnants are cleared or otherwise prevented and that side-channel risks have been assessed. This technical assurance obligation is distinct from deciding which workloads may use the weaker-sharing mode.

## Scope

Applies to any Kubernetes cluster using NVIDIA MPS or GPU time-slicing. Covers GPU memory remnant testing, memory clearing or reset verification, and side-channel assessment activities for shared GPU deployments.

## Enforcement

- Primary Enforcement Point: GPU memory isolation tests, GPU memory clearing controls, and side-channel assessment activities for shared GPU modes.
- Implementation Surface: GPU memory isolation verification tests, GPU reset or clearing mechanisms between workload transitions, side-channel assessment processes for shared GPU deployments, and technical evidence showing no recoverable data remnants across tenant transitions.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Data Protection Office
