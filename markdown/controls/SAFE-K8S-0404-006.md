# SAFE-K8S-0404-006 - Certificate compromise revocation and re-issuance response

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.4 - Certificate Management
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must maintain and exercise incident response procedures for revoking compromised certificates, re-issuing new certificates and key pairs, and validating service recovery after private key compromise.

## Objective

Ensure rapid containment and recovery when certificate private keys are compromised so attackers cannot continue impersonating affected services.

## Rationale

Private key compromise remains high impact even when renewal is automated. A tested revocation and re-issuance procedure reduces the window in which a stolen certificate can be used to impersonate services or intercept traffic.

## Scope

Applies to compromise scenarios affecting API server, etcd, kubelet, webhook, service mesh, and application certificates. Covers revocation, re-issuance, redeployment, and exercise-based validation of response capability.

## Enforcement

- Primary Enforcement Point: Certificate compromise incident response workflows and service recovery validation.
- Implementation Surface: Documented certificate compromise playbooks, revocation and re-issuance procedures, issuance of new key pairs, service redeployment validation, and exercise records demonstrating response execution.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Security Operations
