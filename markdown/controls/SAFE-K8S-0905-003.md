# SAFE-K8S-0905-003 - Model registry hardening

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Model registries (Harbor, MLflow Model Registry, Hugging Face Hub on-prem) must enforce role-based access controls, enable audit logging for all registry operations, apply retention policies for model versions, and secure registry federation and replication channels for multi-cluster and multi-region deployments.

## Objective

Protect model registry integrity and confidentiality by controlling who can publish, access, and manage model versions and by ensuring secure replication across distributed infrastructure.

## Rationale

Model registries are the authoritative source of production models. Without access controls, unauthorized users can publish malicious models or access proprietary model weights. Without audit logging, model substitution attacks go undetected. Without retention policies, storage costs grow unbounded and deprecated models remain accessible. Without secure federation, replication traffic between registries can be intercepted or tampered with.

## Scope

Applies to all model registry deployments on Kubernetes, including Harbor, MLflow Model Registry, and on-premises Hugging Face Hub instances. Covers registry access controls, audit log configuration, retention policy settings, and federation/replication security for multi-cluster environments. Excludes container image registries unless they also store model artifacts.

## Enforcement

- Primary Enforcement Point: Model registry RBAC configuration, audit log settings, retention policy definitions, and federation/replication TLS and authentication settings.
- Implementation Surface: Model registry platforms (Harbor, MLflow Model Registry, Hugging Face Hub on-prem) with RBAC (project-scoped roles, OIDC integration), audit logging with SIEM integration, tag retention policies and garbage collection, registry federation and replication with TLS and endpoint authentication, digest verification after replication, and webhook-based event forwarding for registry operations.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
