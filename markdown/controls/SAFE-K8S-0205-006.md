# SAFE-K8S-0205-006 - Node-level firewall rule restriction and audit

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.5 - kube-proxy and Node Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes nodes must implement host-level firewall rules that restrict inbound and outbound traffic to required cluster communication ports and protocols, with periodic audit and controlled change management for firewall rule modifications.

## Objective

Provide defense-in-depth network segmentation at the node level by enforcing restrictive host firewall or cloud security group rules around authorized cluster communication paths.

## Rationale

Node-level firewall rules provide an independent protection layer even if pod-level policy enforcement is bypassed or misconfigured. This control isolates the firewall restriction and audit obligation so teams can independently verify that node communication is constrained to approved ports and protocols.

## Scope

Applies to all production cluster nodes. Cloud environments may use security groups or VPC firewall rules as the node-level firewall layer. On-premises environments may use iptables, nftables, or firewalld.

## Enforcement

- Primary Enforcement Point: Cloud security group or host firewall rules applied via infrastructure-as-code templates and audited periodically.
- Implementation Surface: Cloud security groups, VPC firewall rules, NSGs, on-premises firewall tools, firewall verification scripts, and firewall change management workflows.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Cloud Infrastructure; Security Engineering
