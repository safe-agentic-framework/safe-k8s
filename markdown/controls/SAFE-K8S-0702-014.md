# SAFE-K8S-0702-014 - Pipeline data classification taxonomy and label propagation

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must define a data type classification taxonomy for pipeline data flows and ensure classification labels are applied and propagated across pipeline resources and stages of data processing.

## Objective

Ensure data sensitivity metadata is consistently assigned and preserved across pipeline resources so downstream controls can reason about data type and sensitivity throughout the processing lifecycle.

## Rationale

Classification-based controls fail when pipeline resources do not carry consistent labels or when labels are lost across stages. A distinct control is needed for the taxonomy itself, the assignment model, and propagation of classification metadata from source artifacts to derived artifacts.

## Scope

Applies to all data pipeline workloads, including ETL jobs, preprocessing steps, feature engineering pipelines, training jobs, and model-serving data paths. Covers taxonomy definition and label propagation across PersistentVolumeClaims, pods, jobs, services, and related pipeline resources.

## Enforcement

- Primary Enforcement Point: Data governance taxonomy and pipeline metadata labeling model.
- Implementation Surface: Organizational data classification taxonomy documentation, PVC and workload metadata labels for data type classification, and orchestrator-level propagation of classification labels across Kubeflow Pipelines, Argo Workflows, and similar pipeline stages.
- Owner Primary: Data Governance
- Owner Supporting: Platform Engineering; Security Engineering
