# SAFE-K8S-0304-005 - Training job ephemeral storage and scratch space security

- Domain: D03 - Workload and Pod Security
- Knowledge Area: 3.4 - Secure Defaults and Resource Constraints
- Maturity: Foundational
- Classification: ai-specific

## Statement

AI training jobs must enforce emptyDir sizeLimit constraints on scratch volumes, must use tmpfs-backed emptyDir for sensitive intermediate data (gradients, partial checkpoints, tokenized training samples), and must apply encryption and access controls to temporary checkpoint storage to prevent data exfiltration or tampering through ephemeral volumes.

## Objective

Secure the ephemeral storage used by AI training jobs for intermediate data, checkpoints, and scratch space, preventing disk exhaustion attacks, sensitive data exposure through node-level filesystem access, and checkpoint tampering.

## Rationale

AI training jobs produce large volumes of intermediate data including model gradients, partial checkpoints, and tokenized training samples that may contain sensitive information. EmptyDir volumes without size limits can fill node disk and trigger eviction of all pods on the node. Disk-backed emptyDir data persists on the node filesystem and can be accessed by other containers or recovered after pod deletion. Checkpoints stored without integrity controls can be tampered with to inject backdoors into models during multi-stage training.

## Scope

Applies to all AI/ML training workloads that use emptyDir volumes for scratch space, intermediate data, or temporary checkpoint storage. Inference workloads using ephemeral scratch space for model loading or response caching are also in scope. Persistent volume-backed checkpoint storage is governed by separate data security controls.

## Enforcement

- Primary Enforcement Point: Pod specification validation through policy engine rules checking emptyDir.sizeLimit and emptyDir.medium fields, combined with namespace- level ResourceQuota for ephemeral storage.
- Implementation Surface: Pod spec emptyDir volume configuration (sizeLimit, medium: Memory for tmpfs), policy engine rules enforcing emptyDir.sizeLimit and emptyDir.medium fields in AI workload namespaces, namespace-level ResourceQuota for ephemeral-storage, encrypted PersistentVolumes via StorageClass parameters (AWS EBS CSI with KMS, GCE PD CSI with CMEK, Azure Disk CSI with encryption), checkpoint integrity verification tooling, and PVC access controls restricting read to training job ServiceAccounts.
- Owner Primary: AI/ML Platform team
- Owner Supporting: Platform Engineering; Security Engineering
