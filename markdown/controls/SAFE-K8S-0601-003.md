# SAFE-K8S-0601-003 - Registry access controls

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Container registries must enforce authenticated access for all push and pull operations, apply role-based authorization restricting write access to authorized CI/CD service accounts and release engineers, enforce network restrictions limiting registry access to trusted sources, and manage image pull secrets through Kubernetes Secrets or an external secrets manager with automatic rotation.

## Objective

Prevent unauthorized image upload, tampering, or exfiltration by enforcing authentication, authorization, and network controls on container registries.

## Rationale

An unprotected registry allows adversaries to push malicious images that may later be pulled and deployed to production clusters. Without authentication and authorization, any actor with network access can overwrite legitimate image tags. Leaked or static image pull secrets further expand the attack window, enabling unauthorized image pulls or supply-chain injection attacks.

## Scope

Applies to all container registries (cloud-managed and self-hosted) used by the organization for storing production, staging, and development images. Includes model-serving image registries and OCI artifact stores. Public mirror registries must be proxied through an authenticated pull-through cache.

## Enforcement

- Primary Enforcement Point: Container registry access control configuration and Kubernetes cluster image pull secret management.
- Implementation Surface: Container registry authentication and authorization configuration (robot accounts, service account tokens), registry RBAC policies restricting push/pull permissions, network firewall rules and CIDR restrictions limiting registry access, image pull secret management via Kubernetes Secrets and External Secrets Operator (ESO), image pull secret automatic rotation configuration, tag immutability settings on production repositories, and digest pinning (image@sha256:...) enforcement in deployment manifests.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
