# SAFE-K8S-0104-002 - Kubernetes version lifecycle and upgrade strategy management

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must run a supported Kubernetes version within the project's active support window, maintain compliance with the Kubernetes version skew policy across all components (API server, kubelet, controller-manager, kubectl), and implement a documented upgrade strategy that includes deprecation tracking, pre-upgrade validation, and rollback procedures.

## Objective

Ensure clusters remain within the supported version window to receive security patches and maintain component compatibility by enforcing version lifecycle management and upgrade planning.

## Rationale

Kubernetes supports only the three most recent minor versions with security patches. Running an unsupported version means known CVEs will not receive patches, leaving the cluster exposed to publicly disclosed vulnerabilities. Version skew between components (e.g., kubelet running a version more than one minor release behind the API server) can cause unexpected behavior, API incompatibilities, and security control failures. Unplanned upgrades without rollback procedures risk control-plane outages.

## Scope

Applies to all production clusters. Covers Kubernetes control-plane and worker node component versions, client tool versions (kubectl), and add-on compatibility. For managed Kubernetes services, applies to selecting supported versions, enabling auto-upgrade channels where appropriate, and tracking provider deprecation timelines.

## Enforcement

- Primary Enforcement Point: API Server / Admission
- Implementation Surface: Cluster version inventory, upgrade planning documentation, version skew monitoring, and deprecation tracking integrated into change management processes.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Change Management
