# SAFE-K8S-0904-004 - Experiment tracking security and pipeline integrity

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0904-009

## Statement

Experiment tracking systems (MLflow, Weights & Biases) must enforce access controls on experiment metadata, hyperparameters, and metrics data, and pipeline definitions must be protected through signed manifests, immutable versions, and rollback controls to prevent unauthorized modification.

## Objective

Protect the confidentiality and integrity of experiment tracking data and ensure that pipeline definitions cannot be tampered with or silently modified.

## Rationale

Experiment metadata includes hyperparameters, performance metrics, and model configurations that reveal proprietary ML techniques and intellectual property. Without access controls, competitors or insider threats can access this data. Pipeline definitions control the entire ML workflow; without signed manifests and immutable versions, an attacker can modify a pipeline to inject backdoors, exfiltrate data, or deploy compromised models without leaving an audit trail.

## Scope

Applies to all experiment tracking platforms and pipeline definition stores deployed on Kubernetes. Covers MLflow tracking server, Weights & Biases server instances, and pipeline definition repositories. Includes RBAC for experiment data and pipeline versioning systems. Excludes model artifact storage covered by KA 9.5.

## Enforcement

- Primary Enforcement Point: Experiment tracking server RBAC configuration, pipeline definition version control with signature verification, and immutable artifact storage settings.
- Implementation Surface: Experiment tracking server RBAC configuration (MLflow authentication and project-level access controls, Weights & Biases server access policies), audit logging for experiment tracking operations, pipeline definition signing using cosign or in-toto attestations, immutable pipeline version storage (Argo WorkflowTemplates, Kubeflow Pipeline YAML), pipeline definition signature verification at execution time via admission control (Kyverno, Gatekeeper), and rollback-capable pipeline version management.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; ML Engineering
