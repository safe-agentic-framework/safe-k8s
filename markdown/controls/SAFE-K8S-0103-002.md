# SAFE-K8S-0103-002 - Scheduler access restriction and extender authentication

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must restrict access to the scheduler API and scheduling decision endpoints to authorized control-plane components only, require authentication and authorization for all scheduler extenders and webhook endpoints, and permit custom schedulers only after security review and approval.

## Objective

Prevent unauthorized influence over workload placement decisions by restricting scheduler access, authenticating scheduler extensions, and governing custom scheduler deployments.

## Rationale

The scheduler determines where workloads run, making it a high-value target for attackers seeking to co-locate malicious pods with sensitive workloads, place pods on compromised nodes, or bypass node-level security controls. Unauthenticated scheduler extenders allow external parties to influence placement decisions. Untrusted custom schedulers can bypass security-aware scheduling constraints entirely.

## Scope

Applies to all production clusters. Covers the default scheduler configuration, scheduler extender webhook endpoints, custom scheduler deployments, and RBAC policies governing scheduler-related resources. For managed Kubernetes services, applies to customer-deployed custom schedulers and scheduler extension configurations.

## Enforcement

- Primary Enforcement Point: RBAC / Authorization
- Implementation Surface: Scheduler configuration and RBAC bindings, scheduler extender webhook TLS and authentication configuration, and custom scheduler deployment approval process.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Application Platform teams
