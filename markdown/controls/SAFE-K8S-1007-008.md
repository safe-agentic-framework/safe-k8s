# SAFE-K8S-1007-008 - GPU cluster scaling governance and node onboarding security

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.7 - Cluster Lifecycle and Asset Inventory
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must enforce security controls for GPU cluster scaling operations including new node onboarding security validation, hardware attestation, driver integrity verification, and GPU node taint verification to ensure that scaling does not introduce unvalidated or compromised nodes.

## Objective

Ensure that GPU infrastructure scaling does not introduce unvalidated or compromised nodes into the trusted compute environment that processes sensitive model data and training workloads.

## Rationale

GPU cluster scaling introduces particular risk: new GPU nodes may have outdated drivers, missing security configurations, or compromised firmware. Without onboarding validation and hardware attestation, scaling operations can introduce untrusted compute capacity. Automated validation gates in the node onboarding pipeline ensure security requirements are met before workloads are scheduled.

## Scope

Applies to all node pool expansion operations that add GPU-capable nodes to Kubernetes clusters hosting AI workloads. Covers node onboarding validation, hardware attestation, driver integrity verification, and GPU taint enforcement. Complements SAFE-K8S-1007-004 (continuous posture management).

## Enforcement

- Primary Enforcement Point: Node onboarding automation pipeline with security validation gates and hardware attestation verification.
- Implementation Surface: GPU node onboarding security validation (kubelet config, container runtime config, OS hardening, GPU driver version verification), hardware attestation for confidential computing (TEE state, firmware integrity), GPU driver integrity verification (checksum comparison against known-good values), and GPU node taint verification (nvidia.com/gpu).
- Owner Primary: Infrastructure Engineering
- Owner Supporting: Platform Engineering; Security Engineering
