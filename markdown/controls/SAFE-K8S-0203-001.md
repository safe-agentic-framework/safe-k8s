# SAFE-K8S-0203-001 - Minimal and immutable host OS configuration

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes node host operating systems must use minimal, purpose-built OS distributions (e.g., Bottlerocket, Flatcar, Talos) or hardened minimal installations with unnecessary packages, services, and user accounts removed, and must enforce immutable root filesystems where supported by the platform.

## Objective

Minimize the host OS attack surface by eliminating unnecessary software components and preventing persistent host-level modifications that could introduce vulnerabilities or backdoors.

## Rationale

General-purpose operating systems include package managers, shell utilities, compilers, and network services that are unnecessary on Kubernetes nodes and provide tools for post-exploitation activities. Each additional package increases the CVE exposure surface and provides utilities an attacker can leverage after gaining initial access. Immutable OS distributions prevent persistent rootkits and make unauthorized host modifications detectable through image drift.

## Scope

Applies to all production cluster node OS images. Cloud-managed node pools must use the provider's hardened or minimal node image options. Custom node images must document their base image, removed packages, and hardening steps. Development clusters may use standard OS images with documented exceptions.

## Enforcement

- Primary Enforcement Point: Node OS image build pipeline, infrastructure-as-code templates, and periodic OS image compliance scanning.
- Implementation Surface: Node OS image build pipeline, minimal OS distributions (Bottlerocket, Flatcar, Talos, Ubuntu Minimal, RHEL CoreOS), immutable root filesystem enforcement (dm-verity), NVIDIA GPU Operator for driver lifecycle, package and service removal automation, and infrastructure-as-code templates.
- Owner Primary: Platform Engineering
- Owner Supporting: OS/Infrastructure; Security Engineering
