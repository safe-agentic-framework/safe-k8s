# SAFE-K8S-1005-010 - AI-specific containment actions for GPU nodes, inference services, and pipeline execution

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.5 - Incident Response for Kubernetes
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must define and document AI-specific containment actions for compromised GPU nodes, inference services, and ML pipeline controllers, including GPU-aware node draining, inference service quarantine, and pipeline execution suspension with state-preservation considerations where applicable.

## Objective

Enable safe containment of AI-specific incidents by preparing targeted GPU, inference, and pipeline isolation actions that preserve evidence and avoid corrupting model training or serving state during response.

## Rationale

AI workloads introduce containment risks beyond standard Kubernetes operations. Draining a GPU node can disrupt co-scheduled training jobs, quarantining inference services can affect customer traffic, and suspending pipelines mid-run can corrupt state or lose evidence. AI-specific containment procedures are required to contain these incidents safely and predictably.

## Scope

Applies to Kubernetes clusters running GPU workloads, model serving platforms, and ML pipeline controllers. Covers containment actions for training jobs, inference services, and orchestration components where AI workload state or shared accelerator resources affect response decisions.

## Enforcement

- Primary Enforcement Point: AI-specific incident response playbooks and service-specific containment procedures for GPU, inference, and pipeline platforms.
- Implementation Surface: GPU-aware drain and quarantine procedures, inference service scale-to-zero or traffic-cutoff procedures, pipeline suspension controls for Argo or equivalent orchestrators, checkpoint-preservation guidance, and AI-specific playbook sections covering these containment actions.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; AI/ML Engineering; SRE
