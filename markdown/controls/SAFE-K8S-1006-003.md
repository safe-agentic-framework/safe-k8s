# SAFE-K8S-1006-003 - Policy-as-code for AI workload compliance

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.6 - Compliance and Governance
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-1006-005

## Statement

Organizations must implement policy-as-code engines (OPA/Gatekeeper, Kyverno, or Kubewarden) to enforce compliance policies on Kubernetes AI workloads, including image provenance verification, GPU resource limit enforcement, and security context constraints, and must configure these engines to generate continuous compliance evidence for audit purposes.

## Objective

Automate enforcement of compliance policies for AI workloads at admission time, eliminating reliance on manual review and producing continuous, machine-readable compliance evidence.

## Rationale

Manual compliance verification for AI workloads does not scale with the frequency of model deployments and training job submissions. Without policy-as-code enforcement, non-compliant AI workloads, such as containers running unsigned model images, training jobs requesting unbounded GPU resources, or inference services with privileged security contexts, can be deployed to production, creating security exposure and compliance violations that are only discovered during periodic audits.

## Scope

Applies to all Kubernetes clusters hosting AI workloads. Policy enforcement must cover AI workload admission, resource allocation, and security context validation. Compliance evidence generation must produce audit-ready artifacts. Policy exceptions must be documented and time-bounded.

## Enforcement

- Primary Enforcement Point: Kubernetes admission controller (OPA/Gatekeeper, Kyverno, or Kubewarden) configuration and policy audit log output.
- Implementation Surface: Policy-as-code engines (OPA/Gatekeeper, Kyverno, Kubewarden), ClusterPolicy resources for image provenance verification (Kyverno verifyImages with cosign/keyless attestors), GPU resource limit enforcement policies (requiring nvidia.com/gpu limits on GPU-enabled namespaces), security context constraint policies, PolicyReport and Gatekeeper audit resources for periodic compliance status reporting, admission decision logging (allow/deny with policy name and rationale), policy version control in Git repositories, policy report and admission decision export to SIEM, and workload-type namespace labels (ai-inference, gpu-enabled) for targeted policy matching.
- Owner Primary: GRC/Compliance
- Owner Supporting: Platform Engineering; AI/ML Engineering; Security Engineering
