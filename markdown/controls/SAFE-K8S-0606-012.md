# SAFE-K8S-0606-012 - Artifact freshness and SCM security posture enforcement

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CI pipelines must enforce artifact freshness limits so deployed artifacts do not exceed the organization-defined maximum age from committed code, and source repositories must be continuously assessed with OpenSSF Scorecard or equivalent to verify source code management security posture before promotion decisions rely on that repository.

## Objective

Reduce trust in stale artifacts and insecure repositories by enforcing freshness limits and automated SCM posture assessment as promotion prerequisites.

## Rationale

Even when code passes security scans, stale artifacts and weak repository controls undermine deployment trust. Build horizon enforcement prevents promotion of code that has aged beyond acceptable patch latency, while repository posture checks catch foundational SCM weaknesses such as missing branch protection or insecure workflows. These are separate audit claims from build-time scanning.

## Scope

Applies to all production deployment pipelines and all source repositories whose outputs may be promoted to production. Includes build horizon policy and automated SCM posture assessment.

## Enforcement

- Primary Enforcement Point: Artifact Promotion / SCM Governance
- Implementation Surface: Build timestamp validation, artifact age enforcement, and automated repository posture assessment tooling such as OpenSSF Scorecard.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Development Teams
