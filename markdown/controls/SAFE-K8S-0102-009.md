# SAFE-K8S-0102-009 - etcd backup storage access restriction and auditing

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must restrict read and write access to etcd backup storage to dedicated backup automation identities and explicitly authorized break-glass administrators only, and must generate auditable records for all access to backup repositories and restore media.

## Objective

Ensure that etcd backups do not become a broadly accessible copy of full cluster state by enforcing least privilege and auditability on backup repositories.

## Rationale

Even when backups are encrypted, excessive repository access expands the attack surface and weakens separation of duties. This control isolates backup repository authorization and access monitoring from encryption and integrity functions so ownership and evidence remain independently auditable.

## Scope

Applies to all production clusters. Covers object storage policies, IAM roles, break-glass access paths, access logging, and restore media handling controls for etcd backup repositories.

## Enforcement

- Primary Enforcement Point: Backup Repository Authorization
- Implementation Surface: Backup storage IAM policies, bucket or repository access policies, break-glass authorization records, and backup access audit logs.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Backup and Recovery Operations
