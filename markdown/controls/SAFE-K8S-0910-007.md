# SAFE-K8S-0910-007 - Security-aware multi-cluster placement and federation isolation

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.10 - Multi-Cluster and Federated AI Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

Multi-cluster AI workload orchestration control planes must verify target cluster health and security posture before workload placement, and must implement circuit-breaker controls that isolate compromised clusters from the federation before new placement or continued workload execution is allowed.

## Objective

Ensure that AI workloads are placed only on clusters that meet current security baselines and that compromised clusters can be rapidly isolated before they affect the wider federation.

## Rationale

A multi-cluster AI platform inherits the weakest-cluster problem. Without security-aware placement, workloads can be routed onto clusters with unresolved critical findings, failed admission controls, unsupported Kubernetes versions, or unsafe GPU driver baselines. Without circuit-breaker isolation, a single compromised cluster can continue receiving workloads or participating in federation actions while it spreads poisoned models, exfiltrates training data, or manipulates inference routing.

## Scope

Applies to organizations operating AI workloads across multiple Kubernetes clusters, whether on-premises, multi-cloud, or hybrid. Covers target-cluster security posture verification prior to workload placement and compromised-cluster isolation after suspected or confirmed compromise. Does not cover orchestration identity federation or centralized audit logging, which are addressed by SAFE-K8S-0910-006.

## Enforcement

- Primary Enforcement Point: Security-aware placement policy definitions, target-cluster posture assessment feeds, and circuit-breaker isolation trigger configuration in the multi-cluster orchestration control plane.
- Implementation Surface: Security-aware placement policies verifying target cluster Kubernetes version support, admission controller status, GPU driver versions, and recent security posture assessments, integration with cluster posture telemetry such as SAFE-K8S-1007-004 outputs, circuit-breaker isolation triggers revoking federation credentials and draining workloads from compromised clusters, manual override and re-admission workflows, and placement decision logs recording the security justification for cluster selection or rejection.
- Owner Primary: Platform Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering; SRE
