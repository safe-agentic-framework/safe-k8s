# SAFE-K8S-0103-007 - Controller-manager and scheduler bind-address and port hardening

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must bind the controller-manager and scheduler to the loopback address (127.0.0.1 or ::1) for their secure serving ports and must disable insecure serving ports (--port=0 or equivalent deprecation handling). The healthz and metrics endpoints must not be exposed on network-accessible interfaces unless explicitly required and protected by authentication.

## Objective

Prevent network-based unauthorized access to controller-manager and scheduler endpoints by ensuring these components only listen on localhost and do not expose unauthenticated ports.

## Rationale

The controller-manager and scheduler expose HTTP endpoints for health checks, metrics, and profiling. Historically, these components served on insecure (unauthenticated) ports accessible on all interfaces. If these endpoints are network-accessible, attackers on the control plane network can extract operational metrics, trigger profiling dumps, or probe health status without authentication. Binding to loopback and disabling insecure ports ensures only local processes (and authenticated secure-port clients) can access these endpoints.

## Scope

Applies to all self-managed Kubernetes clusters. Covers --bind-address, --secure-port, and deprecated --port / --address flags on both kube-controller-manager and kube-scheduler. For managed Kubernetes services, this is provider-managed and should be documented in the shared responsibility model.

## Enforcement

- Primary Enforcement Point: Controller-Manager Configuration
- Implementation Surface: Controller-manager and scheduler static pod manifests or systemd unit configurations, network policy for control plane nodes, CIS Benchmark scanning (sections 1.3 and 1.4).
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering
