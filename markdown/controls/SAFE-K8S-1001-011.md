# SAFE-K8S-1001-011 - Durable and tamper-resistant audit backend delivery

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Production Kubernetes clusters running AI workloads must direct Kubernetes audit records to a durable, tamper-resistant backend that preserves audit data for investigation, compliance, and recovery use.

## Objective

Ensure that captured audit records remain available and trustworthy after collection, including during backend disruption, incident investigation, and evidence review.

## Rationale

An audit policy only creates security value when the resulting records reach storage that resists loss and tampering. Without durable backend delivery, adversaries or operational failures can erase evidence of AI-relevant activity even when audit coverage was configured correctly.

## Scope

Applies to audit record delivery paths and storage backends for Kubernetes clusters hosting AI workloads, including webhook targets, log file destinations, centralized logging platforms, and archival storage.

## Enforcement

- Primary Enforcement Point: Audit log backend configuration and audit record delivery paths.
- Implementation Surface: Kube-apiserver audit webhook configuration, audit log path configuration, durable external storage or SIEM delivery targets, fallback delivery design, and tamper-resistant retention settings for audit records.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; GRC/Compliance
