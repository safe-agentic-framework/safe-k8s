# SAFE-K8S-0905-002 - ML artifact signing and attestation generation

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

All ML lifecycle artifacts (models, training inputs, outputs, intermediate artifacts) must be cryptographically signed using Sigstore or equivalent signing infrastructure, models must be stored as OCI artifacts with digest pinning and tag immutability, and in-toto attestations must be generated for training pipeline steps.

## Objective

Establish cryptographic chain of custody for model artifacts at build time by signing all artifacts and generating pipeline attestations.

## Rationale

Without cryptographic signing, there is no assurance that a model was produced by an authorized pipeline and has not been tampered with. Without digest pinning and tag immutability, a mutable tag can silently point to a different model. Without in-toto attestations, the training pipeline itself cannot be verified.

## Scope

Applies to all model artifacts produced by training pipelines and stored in OCI registries, model registries, and object storage. Covers signing of models, training data manifests, intermediate pipeline artifacts, and final outputs. Covers OCI registry configuration for digest pinning and tag immutability. Excludes deploy-time verification covered by SAFE-K8S-0905-009.

## Enforcement

- Primary Enforcement Point: Model signing pipeline (Sigstore cosign or equivalent) and OCI registry tag immutability settings.
- Implementation Surface: Sigstore cosign keyless signing infrastructure with identity-based trust, in-toto attestation generation for training pipeline steps, OCI registries (Harbor, ECR, GCR, ACR) with digest pinning and tag immutability, deployment manifest references by digest (@sha256:...), and internal re-signing process for externally sourced models.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; DevOps Engineering
