# SAFE-K8S-0606-023 - IaC security scanning gate enforcement for deployment platforms

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Infrastructure as Code definitions for deployment platforms must undergo security scanning in the CI pipeline, and changes that fail required IaC security checks must be blocked from promotion or deployment.

## Objective

Prevent insecure infrastructure templates from reaching staging or production by enforcing IaC scanning as a delivery gate.

## Rationale

Terraform, Pulumi, and CloudFormation templates can weaken network boundaries, expose services, disable encryption, or introduce privileged infrastructure changes. IaC scanning is a distinct preventive control because it evaluates template risk before any infrastructure is created or modified.

## Scope

Applies to Infrastructure as Code used for staging and production platform provisioning, including cluster infrastructure, cloud networking, and supporting security services.

## Enforcement

- Primary Enforcement Point: CI pipeline security gate for infrastructure template changes.
- Implementation Surface: IaC scanning tools and policies integrated into the CI pipeline, with blocking gates for non-compliant Terraform, Pulumi, or CloudFormation changes before platform deployment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
