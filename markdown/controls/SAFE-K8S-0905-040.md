# SAFE-K8S-0905-040 - Model registry audit event generation and centralized forwarding

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Model registries must generate audit logs for model publish, pull, tag, permission, deletion, and administrative operations and must forward those events to centralized monitoring or SIEM platforms with verified delivery and retention.

## Objective

Create reliable telemetry coverage for model-registry abuse by ensuring registry operations are recorded and forwarded into centralized monitoring.

## Rationale

Registry compromises often manifest first as unusual publish, pull, deletion, or permission-change activity. Without complete event generation and forwarding, malicious model substitution, unauthorized access, and registry tampering can occur without durable telemetry or centralized visibility.

## Scope

Applies to all model registry deployments on Kubernetes, including Harbor, MLflow Model Registry, and on-premises Hugging Face Hub instances. Covers registry audit event generation and forwarding into centralized monitoring. Excludes alert triage and operational review procedures.

## Enforcement

- Primary Enforcement Point: Registry audit log configuration and forwarding pipeline into centralized monitoring or SIEM.
- Implementation Surface: Registry audit log enablement, event export or webhook forwarding, log ingestion verification, and delivery checks for sensitive registry operations.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
