# SAFE-K8S-1001-013 - SIEM event forwarding and ingestion health for Kubernetes AI workloads

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must forward Kubernetes audit logs and AI workload security events to a SIEM platform through monitored ingestion pipelines and must continuously verify forwarding health, delivery completeness, and source coverage for the AI security telemetry required by the SOC.

## Objective

Ensure that the SIEM receives complete, timely Kubernetes and AI workload security telemetry so downstream detection and investigation can operate on reliable event streams.

## Rationale

SIEM correlation and alerting are ineffective if the underlying event streams never arrive or silently drop. The ingestion path itself is therefore an auditable control surface: forwarding pipelines, source onboarding, and delivery health determine whether the SOC has the telemetry needed to detect attacks against AI infrastructure.

## Scope

Applies to all production Kubernetes clusters hosting AI workloads. Covers Kubernetes audit log forwarding, AI workload security event forwarding, provider audit source onboarding, and monitoring of SIEM delivery completeness. Excludes the design and maintenance of SIEM correlation logic.

## Enforcement

- Primary Enforcement Point: SIEM ingestion pipeline configuration and log forwarding health monitoring.
- Implementation Surface: Kube-apiserver audit webhook backend forwarding to the SIEM, cloud-provider audit source connectors, GPU and AI workload event forwarding pipelines, pipeline execution event forwarding, and delivery health dashboards or loss-detection checks for the SIEM ingest path.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; AI/ML Engineering; Detection Engineering
