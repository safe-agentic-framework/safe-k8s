# SAFE-K8S-0801-007 - MPS and time-slicing residual-risk acceptance and workload eligibility controls

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Advanced
- Classification: ai-specific
- Superseded By: SAFE-K8S-0801-011

## Statement

Clusters using NVIDIA MPS or GPU time-slicing must document the reduced isolation guarantees compared to MIG, restrict shared-GPU use to approved workload classes and same-trust co-location patterns, and require explicit risk acceptance when sensitive workloads use weaker GPU-sharing modes.

## Objective

Ensure organizations make a conscious governance decision before using weaker GPU-sharing modes and restrict those modes to workload combinations with acceptable trust assumptions.

## Rationale

MPS and time-slicing do not provide the hardware isolation guarantees of MIG. Organizations need a separate control to decide when weaker sharing is acceptable, which workloads may use it, and which compensating controls such as same-trust co-location and admission restrictions must be enforced before deployment.

## Scope

Applies to any Kubernetes cluster using NVIDIA MPS or GPU time-slicing for multi-workload GPU sharing. Sensitive or regulated workloads must not use these modes without documented risk acceptance and workload eligibility restrictions.

## Enforcement

- Primary Enforcement Point: GPU sharing policy, admission restrictions, and workload scheduling eligibility for shared GPU modes.
- Implementation Surface: GPU Operator time-slicing configuration, NVIDIA MPS daemon usage policy, admission control policies restricting sensitive workloads from time-sliced or MPS GPU pools, and node affinity or taint policies enforcing same-trust co-location for shared GPU contexts.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Data Protection Office
