# SAFE-K8S-0102-016 - etcd backup repository access restriction and break-glass authorization

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.2 - etcd and Cluster State Protection
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must restrict read and write access to etcd backup repositories and restore media to dedicated backup automation identities and explicitly authorized break-glass administrators only.

## Objective

Ensure that etcd backups do not become a broadly accessible copy of full cluster state by enforcing least-privilege access and tightly scoped emergency access to backup repositories.

## Rationale

Even when backups are encrypted, excessive repository access expands the attack surface and weakens separation of duties. Restricting routine and emergency access to dedicated identities reduces the risk that backup repositories become a secondary path to full cluster-state exposure.

## Scope

Applies to all production clusters. Covers object storage policies, IAM roles, break-glass access paths, and restore media handling controls for etcd backup repositories.

## Enforcement

- Primary Enforcement Point: Backup Repository Authorization
- Implementation Surface: Backup storage IAM policies, bucket or repository access policies, dedicated backup automation identities, and break-glass authorization records for restore media or repository access.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Backup and Recovery Operations
