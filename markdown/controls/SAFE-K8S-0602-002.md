# SAFE-K8S-0602-002 - Notary v2 (notation) trust policy and signature verification

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.2 - Image Signing and Admission Enforcement
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations using Notary v2 (notation) must configure trust policies that specify trusted signing identities and certificate chains, enforce signature verification for all images pulled from registries, and maintain registry integration that stores signatures as OCI-compliant artifacts alongside their corresponding image manifests.

## Objective

Provide an alternative or complementary image signing and verification mechanism using Notary v2, ensuring trust policy governance over which signing identities are authorized.

## Rationale

Relying on a single signing framework creates a single point of failure and limits interoperability with registries and toolchains that support Notary v2 natively. Without properly configured trust policies, signature verification may accept signatures from unauthorized identities, and without registry integration, signatures may be stored separately from images, creating synchronization and discovery gaps that undermine verification reliability.

## Scope

Applies to all container registries and Kubernetes clusters where Notary v2 is the designated or supplementary signing mechanism. Includes environments using Azure Container Registry, AWS ECR, or other registries with native notation support. Organizations exclusively using Sigstore/cosign may document this control as not applicable.

## Enforcement

- Primary Enforcement Point: Registry-level signature storage configuration and Kubernetes admission controller trust policy enforcement.
- Implementation Surface: Notary v2 notation CLI and trust policy configuration (trustpolicy.json), trusted signing identity and certificate chain specifications, OCI-compliant artifact signature storage alongside image manifests, registry integration for notation signatures (Azure Container Registry, AWS ECR, Harbor), organizational PKI certificate management infrastructure, and notation signature verification at admission via Kubernetes admission controllers.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; DevSecOps / Release Engineering
