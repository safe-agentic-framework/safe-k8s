# SAFE-K8S-1003-003 - MITRE ATT&CK for Containers threat mapping

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.3 - Threat Modeling Methodologies
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must map their Kubernetes AI infrastructure defenses to the MITRE ATT&CK Threat Matrix for Containers, identifying coverage gaps across tactics (initial access, execution, persistence, privilege escalation, defense evasion, credential access, discovery, lateral movement, collection, exfiltration, impact) and must develop detection rules aligned to techniques relevant to AI workload attack scenarios.

## Objective

Validate defensive coverage against known container and Kubernetes attack techniques and ensure detection capabilities are aligned with real-world adversary behavior targeting AI infrastructure.

## Rationale

The MITRE ATT&CK framework for Containers codifies observed adversary tactics and techniques against container environments. Without systematic mapping of existing defenses to this matrix, organizations cannot identify detection blind spots, such as missing coverage for container escape techniques exploited to access GPU resources or persistence mechanisms targeting AI operator controllers.

## Scope

Applies to all Kubernetes clusters hosting AI workloads. Coverage mapping must include both infrastructure-level and AI workload-level detection capabilities. Must be updated when ATT&CK matrix versions are released or when AI infrastructure architecture changes significantly.

## Enforcement

- Primary Enforcement Point: Detection engineering review process, ATT&CK coverage mapping documentation, and detection rule management platform.
- Implementation Surface: MITRE ATT&CK Threat Matrix for Containers coverage mapping documentation, detection engineering review process, detection rule management platform (SIEM correlation rules, EDR rules), tactic-by-tactic defense coverage analysis (initial access, execution, persistence, privilege escalation, defense evasion, credential access, discovery, lateral movement, collection, exfiltration, impact), AI workload-specific technique detection rules (container escape to GPU access, persistence via AI operator controllers), gap remediation plans with timelines, and periodic coverage review aligned with ATT&CK matrix version releases.
- Owner Primary: Security Architecture
- Owner Supporting: Detection Engineering; Security Operations; AI/ML Engineering
