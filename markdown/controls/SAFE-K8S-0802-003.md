# SAFE-K8S-0802-003 - Device plugin socket and kubelet communication security

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.2 - GPU Driver, Library, and Toolkit Security
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0802-006

## Statement

Device plugin Unix domain sockets must be protected with restrictive filesystem permissions, device plugin registration with the kubelet must be authenticated and monitored, and unauthorized GPU access attempts through device plugin interfaces must be detected and blocked.

## Objective

Secure the communication channel between GPU device plugins and the kubelet to prevent unauthorized processes from registering rogue device plugins or intercepting GPU allocation decisions.

## Rationale

GPU device plugins communicate with the kubelet over Unix domain sockets in /var/lib/kubelet/device-plugins/. An attacker with access to this socket directory can register a malicious device plugin that intercepts GPU allocation requests, reports false device health to disrupt scheduling, or redirects workloads to compromised devices. Unmonitored device health reporting can mask hardware failures or tampering that degrades model training integrity.

## Scope

Applies to all Kubernetes nodes where GPU device plugins are registered with the kubelet. Covers the device plugin socket directory, gRPC registration interface, and device health monitoring callbacks. Managed Kubernetes services where kubelet configuration is provider-managed must verify socket permission controls through provider attestation.

## Enforcement

- Primary Enforcement Point: Node-level filesystem permissions on /var/lib/kubelet/device-plugins/, kubelet device plugin registration monitoring, and host intrusion detection rules for unauthorized socket access.
- Implementation Surface: Filesystem permissions on /var/lib/kubelet/device-plugins/ directory and Unix domain sockets (nvidia.sock), kubelet device plugin registration gRPC protocol monitoring, admission control policies (OPA/Gatekeeper, Kyverno) restricting hostPath mounts to the device-plugins directory, runtime security tools (Falco, Tetragon) detecting unauthorized socket creation and rogue plugin registration, device plugin inventory tracking (resource names, socket paths, source images), and host intrusion detection rules for the device-plugins directory.

- Owner Supporting: ML Platform Engineering; Security Engineering
