# SAFE-K8S-0101-010 - API server anonymous authentication disablement and authorization mode hardening

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must disable anonymous API server authentication and must not enable AlwaysAllow authorization on the API server.

## Objective

Reduce unauthorized control-plane access by requiring authenticated API requests and policy-based authorization on the Kubernetes API server.

## Rationale

Anonymous API access and AlwaysAllow authorization bypass the identity and authorization controls that protect the Kubernetes control plane. Requiring authenticated requests and non-bypass authorization modes reduces the risk of unauthorized API access and privilege misuse.

## Scope

Applies to all production Kubernetes clusters and to non-production clusters that host sensitive data, shared workloads, or provide a path to production. For managed Kubernetes services, this control applies to all customer-exposed API server settings and to compensating controls where provider-managed settings are not directly configurable.

## Enforcement

- Primary Enforcement Point: API Server Authentication and Authorization Configuration
- Implementation Surface: Cluster provisioning templates, API server configuration management, managed Kubernetes security settings, and periodic configuration assessment.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
