# SAFE-K8S-0601-013 - Kubernetes image pull secret management and rotation

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Image pull secrets for authenticated registries must be managed through Kubernetes Secrets or an external secrets manager and rotated automatically to reduce the exposure window for leaked registry credentials.

## Objective

Prevent unauthorized registry pulls and long-lived credential exposure by managing image pull secrets through controlled secret distribution and automated rotation.

## Rationale

Leaked or static image pull secrets expand the attack window for unauthorized image pulls and supply-chain injection attacks. Separating pull secret management from registry-side authorization makes it possible to audit credential handling, storage, and rotation independently of registry RBAC policies.

## Scope

Applies to Kubernetes clusters and CI/CD workflows that use authenticated registries for pulling images. Covers image pull secret storage, synchronization from external secret managers, and automatic credential rotation.

## Enforcement

- Primary Enforcement Point: Kubernetes cluster image pull secret lifecycle management.
- Implementation Surface: Image pull secret management via Kubernetes Secrets and External Secrets Operator or equivalent external secret integration, secret synchronization workflows, automatic rotation configuration, and avoidance of embedded registry credentials in Dockerfiles or CI environment variables.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
