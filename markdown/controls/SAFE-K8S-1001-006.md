# SAFE-K8S-1001-006 - SIEM ingestion for Kubernetes AI security events

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must forward Kubernetes audit logs and AI workload security events to a SIEM platform to enable centralized security analysis.

## Objective

Enable real-time centralized security analysis by providing the SIEM with comprehensive Kubernetes audit log and AI workload event data streams.

## Rationale

Kubernetes audit logs and AI workload events require centralized ingestion into a SIEM before any correlation or detection activity is possible. Without a functioning forwarding pipeline, downstream correlation rules cannot operate, and security teams lack the data needed to detect attacks targeting AI infrastructure.

## Scope

Applies to all production Kubernetes clusters hosting AI workloads. Covers audit log forwarding pipelines and SIEM ingestion configuration. Managed Kubernetes services must forward available provider audit logs to the organizational SIEM.

## Enforcement

- Primary Enforcement Point: SIEM ingestion pipeline configuration and log forwarding health monitoring.
- Implementation Surface: Kube-apiserver audit webhook backend forwarding to SIEM, GPU telemetry forwarding from DCGM exporter, Prometheus AI workload metric forwarding (training progress, inference latency, model accuracy), pipeline execution event forwarding, and log forwarding health monitoring dashboards.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; AI/ML Engineering; Detection Engineering
