# SAFE-K8S-0202-006 - Node seccomp defaults and mandatory access control activation

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.2 - Container Runtime Security
- Maturity: Foundational
- Classification: baseline

## Statement

Nodes must enforce default seccomp profiles for container workloads and must activate AppArmor or SELinux mandatory access control on supported nodes, with policy mechanisms ensuring workloads inherit approved kernel security constraints by default unless a documented exception exists.

## Objective

Reduce container escape risk by ensuring node-level kernel security mechanisms are enabled and applied by default to container workloads before runtime vulnerabilities can be exploited.

## Rationale

Default seccomp profiles and mandatory access control systems reduce the kernel attack surface available to container workloads. Even when a workload reaches a vulnerable code path, these controls constrain system calls, file access, and process behavior so that kernel and runtime vulnerabilities are harder to weaponize into host compromise.

## Scope

Applies to production cluster nodes and the workloads scheduled onto them. Seccomp defaults and AppArmor or SELinux activation must be validated for supported operating systems, node pools, and runtime configurations, with exceptions documented and reviewed.

## Enforcement

- Primary Enforcement Point: Node-level kernel security configuration and admission or policy controls that ensure workloads receive approved seccomp and MAC protections by default.
- Implementation Surface: Default seccomp profile configuration, AppArmor profile loading, SELinux enablement and labeling prerequisites, and admission or policy enforcement for workload security context settings related to kernel security profiles.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; OS/Infrastructure
