# SAFE-K8S-0905-049 - OCI model registry tag immutability and overwrite prevention

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

OCI registries that store or distribute models must enforce tag immutability and block tag overwrite or tag reuse so that approved model version tags cannot be retargeted after review.

## Objective

Ensure that OCI registries storing approved models prevent post-review tag mutation or overwrite that could redirect downstream consumers to a different artifact.

## Rationale

Even if consumers are expected to use digests, registries that allow tag overwrite or reuse create an avoidable path for confusion, operational drift, and artifact substitution. Registry-side tag immutability prevents approved model tags from being silently retargeted after release.

## Scope

Applies to OCI registries and model registries backed by OCI artifacts that store approved models. Covers immutable-tag registry settings, blocked overwrite attempts, and audit visibility into tag mutation enforcement.

## Enforcement

- Primary Enforcement Point: OCI registry configuration and audit controls for model artifact tags.
- Implementation Surface: OCI registries such as Harbor, ECR, GCR, or ACR configured for immutable tags, registry policies that block tag overwrite or reuse, and audit records showing tag immutability enforcement for model artifacts.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; DevOps Engineering
