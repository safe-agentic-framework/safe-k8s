# SAFE-K8S-0601-012 - Container registry authentication, authorization, and network restriction

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Container registries must enforce authenticated access for all push and pull operations, apply role-based authorization restricting write access to authorized CI/CD service accounts and release engineers, and enforce network restrictions limiting registry access to trusted sources.

## Objective

Prevent unauthorized image upload, tampering, or exfiltration by enforcing authentication, authorization, and network controls on container registries.

## Rationale

An unprotected registry allows adversaries to push malicious images that may later be pulled and deployed to production clusters. Without authentication, authorization, and network restriction, any actor with network access can overwrite legitimate image tags or exfiltrate proprietary images.

## Scope

Applies to all container registries used for storing production, staging, and development images, including model-serving image registries and OCI artifact stores. Public mirror registries must be proxied through an authenticated pull-through cache.

## Enforcement

- Primary Enforcement Point: Container registry access control configuration.
- Implementation Surface: Container registry authentication configuration, registry RBAC policies restricting push and pull permissions, robot or service account identities, cloud registry IAM policies, network firewall rules and CIDR restrictions, and repository access scope controls.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
