# SAFE-K8S-0505-001 - API server network access restriction

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.5 - API Server and Service Exposure Protection
- Maturity: Practitioner
- Classification: baseline

## Statement

Access to Kubernetes API server endpoints must be restricted to authorized networks using private cluster configurations, firewall rules, or authorized network allowlists, and administrative access must be routed through bastion hosts or VPN connections rather than directly over the public internet.

## Objective

Minimize the network exposure of the Kubernetes API server to reduce the attack surface for credential-based and exploitation attacks against the control plane.

## Rationale

The Kubernetes API server is the most critical control-plane component and a high-value target. Internet-exposed API servers are subject to brute-force authentication attacks, exploitation of API server vulnerabilities, and credential stuffing using leaked kubeconfig files. Even with strong authentication, reducing network reachability eliminates entire classes of attacks and limits the blast radius of credential compromise to actors with network access.

## Scope

Applies to all production Kubernetes clusters. Managed Kubernetes services must use private endpoint or authorized network features. Self-managed clusters must place API servers on private networks with bastion or VPN access. Non-production clusters with access to production secrets or infrastructure must also restrict API server access.

## Enforcement

- Primary Enforcement Point: Cloud provider private cluster configuration, API server authorized network settings, VPC firewall rules, and bastion host access controls.
- Implementation Surface: Cloud provider private cluster configurations (GKE enable-private-nodes/enable-private-endpoint/master-authorized-networks, EKS endpointPublicAccess=false/endpointPrivateAccess=true, AKS enable-private-cluster/private-dns-zone), API server --bind-address and --advertise-address flags for self-managed clusters, VPC firewall rules and cloud security groups restricting port 6443, bastion host deployments with MFA-protected SSH access, VPN connections for administrative kubectl access, SSH tunneling to API server endpoints, and external network scanning to confirm API server unreachability from the public internet.
- Owner Primary: Platform Engineering
- Owner Supporting: Network Engineering; Security Engineering; Cloud Platform Engineering
