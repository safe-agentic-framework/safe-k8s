# SAFE-K8S-0905-018 - ML framework and CUDA dependency vulnerability management

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific
- Superseded By: SAFE-K8S-0905-028

## Statement

Organizations must perform software composition analysis and vulnerability triage for ML frameworks, CUDA toolkits, and Python ecosystem dependencies used in training and inference environments, and must track remediation or compensating actions for identified ML dependency vulnerabilities.

## Objective

Reduce exploitability of vulnerable ML software components by continuously identifying, prioritizing, and remediating defects in frameworks, accelerators, and supporting package dependencies.

## Rationale

ML workloads depend on large dependency graphs across frameworks, GPU libraries, and Python packages. Without focused vulnerability management, known defects in those components remain deployable and exploitable across training and inference environments.

## Scope

Applies to all ML software dependencies used in training and inference environments on Kubernetes. Covers ML frameworks, CUDA toolkits, cuDNN, NCCL, Python packages, and transitive dependencies. Excludes model provenance record generation and deployment-time signature verification.

## Enforcement

- Primary Enforcement Point: CI/CD and image-build SCA scanning, dependency inventory review, and remediation governance.
- Implementation Surface: SCA scanning for PyTorch, TensorFlow, JAX, CUDA, cuDNN, NCCL, and Python dependencies; vulnerability triage workflows; dependency update records; compensating-control approvals; and tracking of unsafe deserialization or dependency-related exposure in ML libraries.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; DevOps Engineering
