# SAFE-K8S-0504-004 - L7 microsegmentation and API-aware policies

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.4 - Zero Trust Architecture and Service Mesh
- Maturity: Practitioner
- Classification: baseline

## Statement

Microsegmentation must extend beyond L3/L4 network policies to include L7 authorization that restricts access based on HTTP method, path, and headers; service-level access control must be enforced at the application protocol layer; and API-aware security policies must validate request patterns against expected service contracts.

## Objective

Achieve fine-grained access control at the application protocol layer to prevent authorized network connections from being used for unauthorized operations.

## Rationale

L3/L4 network policies only control which pods can establish connections, not what operations are performed over those connections. An attacker with access to an authorized pod can use allowed network paths to invoke privileged API endpoints, exfiltrate data via allowed HTTP connections, or exploit services through unexpected HTTP methods. L7 microsegmentation closes this gap by enforcing what operations are permitted on each authorized communication path.

## Scope

Applies to production clusters running a service mesh or CNI with L7 policy support. Priority should be given to services handling sensitive data, admin operations, or cross-trust-boundary communication. L7 policies require application protocol awareness and may not apply to non-HTTP workloads without protocol-specific support.

## Enforcement

- Primary Enforcement Point: Service mesh authorization policies (Istio AuthorizationPolicy with HTTP rules, Cilium L7 policies), API gateway policy enforcement, and envoy filter configurations.
- Implementation Surface: Service mesh L7 authorization policies (Istio AuthorizationPolicy with HTTP method/path/header rules, Linkerd HTTPRoute with per-route AuthorizationPolicy, Cilium CiliumNetworkPolicy L7 HTTP rules), API gateway policy enforcement (request pattern validation against OpenAPI specs), Envoy filter configurations, and L7 policy testing verifying allowed vs. blocked HTTP methods and paths on authorized network connections.
- Owner Primary: Security Engineering
- Owner Supporting: Network Engineering; Application Development
