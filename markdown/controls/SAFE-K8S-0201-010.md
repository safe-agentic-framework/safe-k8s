# SAFE-K8S-0201-010 - Node-level kubelet audit verification

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.1 - Kubelet and Node Configuration Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Each cluster node must configure the Linux audit subsystem or equivalent host audit telemetry to detect read and write access to kubelet configuration files, kubeconfig files, and PKI material, and audit log forwarding must ensure those events reach the centralized audit pipeline.

## Objective

Detect unauthorized access to or modification of kubelet configuration and credential material on cluster nodes through host-level audit telemetry.

## Rationale

File permissions and systemd hardening prevent many misuse paths, but they do not provide a host-level detection record when access occurs through legitimate elevated processes, container escapes, or local compromise. Node-level audit verification is an independent detection control and should be auditable separately from preventative kubelet hardening.

## Scope

Applies to all cluster nodes. Covers auditd or equivalent host audit rules for kubelet configuration and credential paths, audit pipeline forwarding, and recurring validation that those rules remain active. Managed Kubernetes services may satisfy this through provider-documented equivalent node audit telemetry.

## Enforcement

- Primary Enforcement Point: Node Audit Telemetry for Kubelet Configuration and Credential Access
- Implementation Surface: Audit rule configuration, host audit daemon settings, centralized log forwarding, and recurring validation of node-level kubelet access monitoring.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; OS/Infrastructure
