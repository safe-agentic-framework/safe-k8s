# SAFE-K8S-0104-008 - Production feature gate approval and risk assessment governance

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must maintain an inventory of all non-default feature gates, must prohibit alpha-stage feature gates in production unless explicitly risk-assessed and approved, and must perform documented security impact assessment before enabling beta-stage feature gates in production.

## Objective

Govern the enablement of non-default Kubernetes feature gates so unstable or insufficiently assessed functionality is not introduced into production clusters without explicit security review and approval.

## Rationale

Alpha and beta feature gates change cluster behavior in ways that may not be fully understood or stable. Without an explicit inventory and approval process, experimental behavior can accumulate in production clusters and undermine security review, supportability, and multi-tenant predictability.

## Scope

Applies to all production clusters and all customer- or operator-configurable feature gates on API server, controller-manager, scheduler, kubelet, kube-proxy, and managed service surfaces where feature gate behavior can be influenced or must be tracked.

## Enforcement

- Primary Enforcement Point: Cluster provisioning and change approval processes governing non-default feature gate enablement.
- Implementation Surface: Feature gate inventory records, provisioning templates, change approval workflows, and documented risk assessments for alpha or beta gate enablement decisions.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Architecture Review
