# SAFE-K8S-0902-007 - Infrastructure-layer prompt injection defense and token output filtering

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.2 - Inference Server and Model Serving Security
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0902-010

## Statement

LLM serving infrastructure must deploy infrastructure-layer prompt injection defenses (input classification, instruction hierarchy enforcement, and system/user prompt boundary validation) and token-level output filtering (streaming token scanners for PII, secrets, and policy-violating content) that operate independently of the model's own safety training.

## Objective

Provide defense-in-depth against prompt injection attacks and sensitive data leakage at the infrastructure layer, complementing model-level safety training with serving-infrastructure controls that are significantly harder to bypass through prompt manipulation than model-level defenses alone.

## Rationale

Model-level safety training (RLHF, constitutional AI) is the primary defense against prompt injection and harmful outputs, but it is probabilistic and bypassable through novel attack techniques. Infrastructure-layer defenses operate outside the model's context window, making them significantly more resistant to prompt-based bypass techniques. Input classifiers can detect known injection patterns before the prompt reaches the model. Instruction hierarchy enforcement ensures system prompts cannot be overridden by user content. Token-level output scanners inspect the generation stream in real time, catching PII, secrets, or policy violations that the model's own filters miss, including in streaming responses where post-hoc filtering is too late.

## Scope

Applies to all LLM inference endpoints deployed on Kubernetes. Covers preprocessing defenses (prompt injection classifiers, boundary validators) and postprocessing defenses (streaming token-level output scanners). Excludes model training-time safety alignment, which is outside infrastructure scope. Complements SAFE-K8S-0902-003 (general request validation and response filtering) with LLM-specific controls.

## Enforcement

- Primary Enforcement Point: Inference preprocessing middleware or sidecar containers for prompt injection classification, LLM serving framework instruction hierarchy configuration, and streaming output filter middleware or sidecar proxies.
- Implementation Surface: Preprocessing sidecar containers or middleware deploying prompt injection classifiers, LLM serving framework instruction hierarchy configuration (system/user/assistant role enforcement), API gateway schema validation for system/user prompt boundary enforcement, streaming token-level output filter middleware or sidecar proxies, PII and secrets detection regex and NER models in the output pipeline, content-policy classifiers for output token windows, and red team bypass testing infrastructure.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Application Security
