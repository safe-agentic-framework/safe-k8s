# SAFE-K8S-0904-006 - Pipeline stage isolation and cross-classification authorization

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0904-011

## Statement

Pipeline steps must be isolated to prevent data leakage between stages of different sensitivity levels, and data transfers between pipeline stages with different classification levels must require explicit authorization through pipeline-level authorization gates.

## Objective

Prevent data leakage between pipeline stages operating at different sensitivity levels and enforce authorization for cross-classification data movement.

## Rationale

Without stage isolation, a low-sensitivity pipeline step can read artifacts from a high-sensitivity stage. Without authorization gates on cross-classification transfers, data classification boundaries are bypassed, potentially exposing restricted data to unauthorized pipeline components.

## Scope

Applies to all ML pipeline orchestration on Kubernetes where pipeline stages operate at different data classification levels. Covers namespace-level stage isolation, authorization controls for cross-stage and cross-classification data movement, and data leakage testing between stages. Excludes artifact encryption and storage access controls covered by SAFE-K8S-0904-002.

## Enforcement

- Primary Enforcement Point: Pipeline orchestrator step isolation settings and data classification gateway policies for cross-stage transfers.
- Implementation Surface: Data classification gateway policies for cross-stage transfers, admission control or pipeline-level authorization gates for cross-classification data movement, namespace isolation between pipeline stages of different classification levels, and network policies restricting cross-stage communication.
- Owner Primary: Data Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
