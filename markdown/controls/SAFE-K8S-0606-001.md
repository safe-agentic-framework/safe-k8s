# SAFE-K8S-0606-001 - CNCF lifecycle security and zero-trust CI/CD pipelines

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CI/CD pipelines must apply security controls across all CNCF lifecycle phases (Develop, Distribute, Deploy), must establish and verify trust at every pipeline step through artifact integrity verification and authenticated handoffs between stages, and must generate evidence concurrently with build execution that has higher trust than the build process itself, consistent with NIST SP 800-204D zero-trust CI/CD principles.

## Objective

Ensure that the CI/CD pipeline itself is treated as an untrusted component, with independently verifiable evidence generated at each stage to provide supply chain assurance that does not depend solely on pipeline integrity.

## Rationale

Traditional CI/CD pipelines implicitly trust their own outputs: if the pipeline says an image passed scanning, that claim is accepted without independent verification. A compromised pipeline can falsify results. Zero-trust CI/CD per NIST SP 800-204D requires that evidence (signatures, attestations, scan results) be generated by independent components and verified at each transition, so that pipeline compromise does not automatically compromise all downstream assurance.

## Scope

Applies to all CI/CD pipelines that produce, promote, or deploy artifacts to Kubernetes clusters. Covers the full lifecycle from source commit through build, test, registry promotion, and cluster deployment. Managed CI/CD services must be evaluated for equivalent zero-trust evidence generation capabilities.

## Enforcement

- Primary Enforcement Point: CI/CD pipeline orchestration configuration and inter-stage verification gates.
- Implementation Surface: CI/CD pipeline orchestration configuration spanning CNCF Develop/Distribute/Deploy lifecycle phases, inter-stage artifact integrity verification and digest matching, authenticated handoff mechanisms between pipeline stages (OIDC identity tokens, service account authentication), independently generated evidence artifacts (signatures, attestations, scan reports) at each pipeline stage per NIST SP 800-204D zero-trust CI/CD principles, and pipeline configuration ensuring evidence generation components are independent of and have higher trust than the build process itself.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Platform Engineering
