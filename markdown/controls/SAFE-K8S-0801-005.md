# SAFE-K8S-0801-005 - GPU topology metadata protection and admission control

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.1 - GPU Device Plugins and Resource Allocation
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Kubernetes node labels, device plugin advertisements, and topology information exposing GPU hardware details must be restricted to prevent metadata leakage, and admission control policies must validate GPU resource requests to prevent unauthorized or excessive accelerator allocation.

## Objective

Prevent attackers from using Kubernetes-exposed GPU topology metadata to fingerprint hardware configurations, plan targeted attacks, or bypass resource allocation controls through crafted GPU requests.

## Rationale

Device plugin advertisements and GPU Feature Discovery node labels expose detailed hardware topology information including GPU model, driver version, MIG capabilities, NUMA affinity, and interconnect layout. Attackers can use this metadata to identify high-value targets, craft hardware-specific exploits, or schedule malicious workloads onto specific GPU nodes. Without admission control validation, pods can request GPU resources beyond their authorization or manipulate scheduling to co-locate with target workloads.

## Scope

Applies to all Kubernetes clusters with GPU nodes where device plugin advertisements or GPU Feature Discovery labels are active. Includes managed Kubernetes services where node label visibility is configurable. Clusters with uniform GPU hardware and single-tenant usage may reduce label restriction requirements with documented justification.

## Enforcement

- Primary Enforcement Point: GPU Feature Discovery label configuration, RBAC policies restricting node label visibility, and admission control (OPA/Gatekeeper or Kyverno) policies validating GPU resource requests against authorized limits.
- Implementation Surface: GPU Feature Discovery (GFD) label configuration controlling exposed node labels (nvidia.com/gpu.product, nvidia.com/gpu.memory, nvidia.com/gpu.count, nvidia.com/mig.strategy, nvidia.com/gpu.driver.version), Kubernetes RBAC policies restricting node object read access, admission control policies (OPA/Gatekeeper, Kyverno) validating GPU resource requests against namespace-level authorization and ResourceQuota limits, and device plugin resource advertisement filtering to suppress sensitive topology details.

- Owner Supporting: Platform Engineering; Security Engineering
