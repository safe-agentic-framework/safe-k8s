# SAFE-K8S-0910-022 - Cross-cluster communication authorization policy enforcement

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.10 - Multi-Cluster and Federated AI Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

Cross-cluster network communication for distributed AI workloads must enforce authorization policies that restrict communication to explicitly authorized workload pairs, services, and cluster endpoints across federated clusters.

## Objective

Prevent unauthorized cross-cluster AI communication by enforcing explicit peer authorization across federated cluster boundaries.

## Rationale

Encrypted transport and authenticated endpoints do not prevent any federated workload from talking to every other remote workload. Cross-cluster authorization policies constrain reachable peers and limit misuse if an otherwise legitimate federated identity attempts unauthorized communication.

## Scope

Applies to all cross-cluster network communication for AI workloads, including distributed training gradient exchange, multi-cluster inference routing, and cross-cluster pipeline data transfer. Covers authorization rules and explicit allowlists for workload pairs, services, and endpoints across clusters. Excludes traffic anomaly detection and monitoring after communication is permitted.

## Enforcement

- Primary Enforcement Point: Cross-cluster authorization policy enforcement and default-deny peer restrictions.
- Implementation Surface: Cross-cluster AuthorizationPolicy or Cilium ClusterMesh network policies with default-deny and explicit allow rules, service-identity-based allowlists for authorized workload pairs, and policy definitions governing allowed cross-cluster communication paths.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Engineering; ML Platform Engineering
