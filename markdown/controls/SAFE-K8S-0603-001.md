# SAFE-K8S-0603-001 - Artifact attestation with in-toto and SLSA provenance

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.3 - Attestation, Provenance, and Cryptographic Assurance
- Maturity: Advanced
- Classification: baseline

## Statement

All container images promoted to production must carry in-toto attestations that include SLSA provenance metadata and build-time metadata. Attestations must be generated during the CI/CD build process, attached to container images as signed OCI artifacts, and verified before deployment. SBOMit or equivalent tooling must bind SBOM attestations to the image digest.

## Objective

Provide verifiable, tamper-evident metadata about how each container image was built, what inputs were consumed, and what build process was used, enabling supply chain integrity verification.

## Rationale

Image signatures alone prove who signed an artifact but not how it was built or from what source. Without attestations, an authorized signer could sign a compromised image built from tampered source code or injected dependencies. In-toto attestations and SLSA provenance bind the image to its source inputs, build steps, and build environment, enabling detection of supply chain tampering that signature-only verification would miss.

## Scope

Applies to all container images in production and staging registries. Includes model-serving images, training images, and infrastructure operator images. Third-party vendor images must have attestations verified or compensating controls documented.

## Enforcement

- Primary Enforcement Point: CI/CD pipeline attestation generation step and admission controller attestation verification policy.
- Implementation Surface: in-toto attestation framework, SLSA provenance metadata generation (builder identity, source repository, commit SHA, build configuration), cosign attest CLI (--predicate, --type slsaprovenance), OCI artifact attachment of attestations to image digests, SBOMit tooling for binding SBOM attestations to image digests, Kyverno verifyImages attestation blocks for admission-time attestation verification, and SLSA level enforcement policies per environment (SLSA 1/2/3).
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Platform Engineering
