# SAFE-K8S-0904-005 - Scheduled job security and feature freshness monitoring

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Kubernetes CronJobs and batch jobs used for scheduled feature computation and data refresh must follow security hardening standards for job execution, and feature freshness and integrity monitoring must be implemented to detect stale, poisoned, or tampered features before they reach training or inference.

## Objective

Ensure that scheduled ML data processing jobs execute securely and that feature data anomalies caused by staleness, poisoning, or tampering are detected before impacting model quality or security.

## Rationale

Scheduled CronJobs that compute features and refresh data run with elevated data access privileges and are often overlooked in security hardening. Without job security controls, a compromised CronJob can exfiltrate data or inject poisoned features on a schedule. Without freshness and integrity monitoring, stale or tampered features silently degrade model performance or introduce exploitable biases, and the impact is not detected until model behavior visibly fails.

## Scope

Applies to all Kubernetes CronJobs and batch Jobs performing scheduled feature computation, data refresh, or data preprocessing for ML workloads. Covers job security context, RBAC, and secret management, as well as feature freshness monitoring and integrity checks. Excludes ad-hoc training jobs covered by KA 9.1.

## Enforcement

- Primary Enforcement Point: Kubernetes CronJob and Job specifications, feature store freshness monitoring configuration, and feature integrity validation hooks in training and inference pipelines.
- Implementation Surface: Kubernetes CronJob and Job specifications with security context hardening (readOnlyRootFilesystem, non-root, dropped capabilities), dedicated service accounts with minimum data access, activeDeadlineSeconds on Jobs, RBAC restricting CronJob creation and modification, CronJob definition change monitoring, feature freshness timestamp tracking and alerting, statistical drift detection on feature values (Evidently AI, Great Expectations), and feature integrity checksums with hash verification before consumption.
- Owner Primary: Data Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
