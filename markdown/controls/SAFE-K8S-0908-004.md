# SAFE-K8S-0908-004 - Differential privacy and output perturbation

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.8 - Model Abuse and Extraction Prevention
- Maturity: Advanced
- Classification: ai-specific

## Statement

Inference endpoints must deploy differential privacy mechanisms and output perturbation controls to limit information leakage from model responses.

## Objective

Mathematically bound the information an adversary can extract about individual training records from model outputs through calibrated noise addition and output perturbation techniques.

## Rationale

Without differential privacy or output perturbation, each inference response leaks information about the training data, and the cumulative effect of many queries enables effective membership inference and model inversion attacks. Differential privacy provides a formal mathematical guarantee bounding this leakage per inference response. Secure aggregation for aggregate queries is governed by SAFE-K8S-0908-006, which splits the aggregation obligation into its own atomic control.

## Scope

Applies to inference endpoints on Kubernetes serving models trained on sensitive data where privacy guarantees are required. Covers output perturbation configuration and differential privacy budget (epsilon) management for individual inference responses. Excludes training-time differential privacy, which is a model development concern. Secure aggregation for aggregate queries is addressed in SAFE-K8S-0908-006.

## Enforcement

- Primary Enforcement Point: Inference server output perturbation configuration, differential privacy budget tracking middleware, and secure aggregation service configuration.
- Implementation Surface: Inference server output perturbation configuration (calibrated noise addition to confidence scores and probability distributions), differential privacy parameter settings (epsilon, delta) per model, privacy budget tracking middleware with per-principal cumulative spend monitoring, LLM output temperature adjustment and output-level perturbation, and DP utility impact monitoring with alerting on utility degradation thresholds.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Privacy Engineering
