# SAFE-K8S-1001-003 - Centralized logging for AI workloads

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must centralize AI workload logs, including training job logs, inference request and response logs, and pipeline execution logs, into the enterprise logging platform, and must apply PII redaction or masking to inference request and response logs before storage to prevent unauthorized exposure of sensitive data.

## Objective

Enable unified visibility into AI workload activity while preventing the inadvertent persistence of personally identifiable information in log repositories.

## Rationale

AI workloads generate specialized log streams, including training convergence data, inference inputs and outputs, and pipeline orchestration events, that are not captured by standard application logging. Without centralized collection, security teams cannot detect anomalies such as training data poisoning or inference abuse. Without PII redaction, inference logs containing user-submitted data create regulatory liability and expand the blast radius of a log repository compromise.

## Scope

Applies to all AI/ML workloads running on Kubernetes, including training jobs, inference services, and ML pipeline controllers. PII redaction requirements apply specifically to inference request/response logs. Exceptions for non-redacted logging require documented approval and compensating access controls.

## Enforcement

- Primary Enforcement Point: Log pipeline processing rules, PII redaction filters in the log aggregation layer, and centralized logging platform ingestion configuration.
- Implementation Surface: Log pipeline processing rules and PII redaction filters in the log aggregation layer (Fluent Bit Lua filter scripts, Vector transforms), pattern-based redaction (regex for SSN, email, phone, credit card), NER-based redaction for unstructured text in inference request/response logs, field-level masking for structured JSON payloads, training job log labeling (log_class: training), pipeline execution log correlation (pipeline_run_id field), and centralized logging platform ingestion configuration for AI workload log streams (training convergence data, inference inputs/outputs, pipeline orchestration events).
- Owner Primary: Security Operations
- Owner Supporting: AI/ML Engineering; Data Privacy
