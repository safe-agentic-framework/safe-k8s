# SAFE-K8S-1005-005 - Post-incident model integrity verification and recovery decisions

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.5 - Incident Response for Kubernetes
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must verify model integrity after any infrastructure compromise that could have affected AI workloads and must make documented retraining or rollback decisions based on integrity verification results.

## Objective

Ensure that AI models are not silently compromised during infrastructure incidents and that recovery decisions are based on verified model integrity.

## Rationale

An infrastructure compromise that touches AI workloads may result in model tampering (weight modification, backdoor insertion) that is not detectable through standard infrastructure recovery checks. Without post-incident model validation, organizations may restore and serve compromised models.

## Scope

Applies to all AI workloads in Kubernetes clusters affected by a security incident. Covers model integrity verification, retraining decision frameworks, and rollback procedures.

## Enforcement

- Primary Enforcement Point: Incident response recovery procedures and model integrity verification tooling.
- Implementation Surface: Model signature verification tooling (cosign/Sigstore), model registry checksum comparison, behavioral validation against reference datasets for statistical divergence detection, model quarantine and rollback procedures (InferenceService spec update to previous model version), and post-compromise elevated monitoring configuration (30-day accuracy comparison).
- Owner Primary: Security Operations
- Owner Supporting: AI/ML Engineering; Digital Forensics
