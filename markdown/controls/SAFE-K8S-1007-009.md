# SAFE-K8S-1007-009 - Kubernetes API deprecation monitoring and usage discovery

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.7 - Cluster Lifecycle and Asset Inventory
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must monitor official Kubernetes release channels and changelog entries for API deprecations and removals, maintain a registry of deprecated Kubernetes APIs used by cluster workloads, operators, custom resources, and manifests, and notify responsible teams in time to complete required migrations before production upgrades.

## Objective

Identify deprecated Kubernetes API usage affecting AI workloads early enough to plan and complete migrations before removed APIs cause upgrade failures.

## Rationale

Kubernetes removes deprecated APIs on a defined schedule. AI operators, training controllers, inference manifests, and custom resources can continue to reference APIs that are already on a removal path. Continuous deprecation monitoring and usage discovery ensure teams know what must be migrated before an upgrade window begins.

## Scope

Applies to all Kubernetes clusters scheduled for version upgrades. Discovery must cover AI workload namespaces, operator deployments, Helm chart templates, and custom resource definitions used by AI platforms and services.

## Enforcement

- Primary Enforcement Point: Deprecation scanning tools in CI/CD pipelines, release-note monitoring workflows, and migration tracking dashboards.
- Implementation Surface: Monitoring official Kubernetes release notes and CHANGELOG entries, API deprecation scanning tools such as kubent, pluto, kubepug, or kubectl deprecations, manifest and Helm chart scanning for deprecated APIs, CRD deprecated field detection, registry generation for deprecated API usage, and team notification workflows for required migrations.
- Owner Primary: Platform Engineering
- Owner Supporting: AI/ML Engineering; DevOps; Security Engineering
