# SAFE-K8S-0606-019 - Helm chart provenance and signature verification

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Helm charts used for Kubernetes deployments must have verified provenance and validated signatures before installation or reconciliation, and externally sourced charts must be accepted only from approved repositories after authenticity validation.

## Objective

Prevent deployment of tampered or spoofed Helm charts by requiring provenance and signature verification before charts are installed or reconciled.

## Rationale

A Helm chart can fully define privileged Kubernetes resources, hooks, and dependent sub-charts. If teams install charts without verifying provenance and signatures, an attacker can substitute a malicious chart or repository source and obtain execution inside the cluster. Independent provenance validation is therefore a distinct control from dependency governance or values-hardening.

## Scope

Applies to Helm charts used in staging and production Kubernetes environments, including charts for AI platforms, operators, and application workloads. Covers internal and external chart sources.

## Enforcement

- Primary Enforcement Point: Helm chart verification step in CI/CD and GitOps reconciliation before installation.
- Implementation Surface: Helm provenance files (.prov), helm verify execution, chart signature validation in CI/CD or GitOps workflows, and repository acceptance controls for external charts prior to use.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Platform Engineering; Security Engineering
