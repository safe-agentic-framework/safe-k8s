# SAFE-K8S-0605-004 - Admission webhook security and availability

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.5 - Admission Control
- Maturity: Practitioner
- Classification: baseline

## Statement

Admission webhooks must be configured with a failure policy of Fail (fail-closed) for security-critical webhooks, must set appropriate timeout values to prevent API server latency degradation, must use TLS certificates managed through automated rotation (cert-manager or equivalent), and must be deployed with high-availability configurations to prevent webhook unavailability from blocking cluster operations.

## Objective

Ensure admission webhooks are securely configured, resilient to failure, and do not degrade cluster availability while maintaining fail-closed security enforcement.

## Rationale

An admission webhook configured with failurePolicy: Ignore silently stops enforcing policy when the webhook is unavailable, creating a security gap that adversaries can exploit by causing webhook downtime. Expired or misconfigured TLS certificates cause webhook failures that block all API server operations. Without proper timeout configuration, slow webhooks cascade into API server timeouts affecting cluster-wide operations. Balancing fail-closed security with availability requires deliberate configuration.

## Scope

Applies to all validating and mutating admission webhooks in production and staging Kubernetes clusters, including webhooks for image signing, policy enforcement, and resource validation. Covers both custom webhooks and those deployed by admission controller frameworks (Gatekeeper, Kyverno, Kubewarden).

## Enforcement

- Primary Enforcement Point: Kubernetes API server webhook configuration and webhook deployment infrastructure.
- Implementation Surface: Kubernetes ValidatingWebhookConfiguration and MutatingWebhookConfiguration resources (failurePolicy: Fail, timeoutSeconds <= 10), webhook backend Deployment replicas with PodDisruptionBudgets and pod anti-affinity rules, cert-manager Certificate resources for automated TLS certificate rotation and CA injection, namespace exclusion rules for kube-system and webhook namespaces to prevent bootstrap deadlocks, and webhook latency/error rate monitoring dashboards with alerting on p99 > 5s or error rate > 1%.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; SRE / Operations
