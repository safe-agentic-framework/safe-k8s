# SAFE-K8S-0601-002 - Container image vulnerability scanning

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

All container images must undergo automated vulnerability scanning at build time in the CI/CD pipeline, at push time in the container registry, and periodically at runtime. Scanning must cover AI-specific dependencies including PyTorch, TensorFlow, JAX, ONNX Runtime, Triton Inference Server, and their native library chains (CUDA, cuDNN, NCCL). Images with unmitigated critical or high-severity vulnerabilities must not be promoted to production without a documented risk acceptance.

## Objective

Detect and remediate known vulnerabilities in container images across the full lifecycle, including the deep native dependency trees unique to AI/ML frameworks.

## Rationale

AI workload images contain extensive native library chains (CUDA, cuDNN, NCCL) that are not covered by standard language-level dependency scanners. Without scanning that explicitly targets these AI-specific dependencies, critical vulnerabilities in GPU drivers or ML framework binaries remain undetected, exposing inference and training workloads to remote code execution and data exfiltration.

## Scope

Applies to all container images in CI/CD pipelines, container registries, and running workloads across all Kubernetes clusters. Includes both standard application images and AI/ML workload images. Managed service provider base images must be scanned upon import.

## Enforcement

- Primary Enforcement Point: CI/CD pipeline build and test stages, container registry webhook scanning, and runtime container security agents.
- Implementation Surface: CI/CD pipeline vulnerability scanning integration (Trivy, Grype, Snyk), container registry webhook scanning (Harbor, Artifact Registry, ECR, ACR), runtime container security scanning agents (KubeClarity, Trivy Operator), AI-specific native library CVE indexing for CUDA toolkit libraries (libcudart, libcublas, libcufft), cuDNN (libcudnn), NCCL (libnccl), TensorRT (libnvinfer), and Python wheel-bundled native library detection for PyTorch, TensorFlow, JAX, ONNX Runtime, and Triton Inference Server dependencies.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
