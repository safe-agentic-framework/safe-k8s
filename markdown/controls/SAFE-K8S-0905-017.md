# SAFE-K8S-0905-017 - Model registry operation audit logging and forwarding

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0905-040

## Statement

Model registries must generate audit logs for model publish, pull, tag, permission, deletion, and administrative operations and must forward those events to centralized monitoring so abnormal registry activity can be investigated and alerted on.

## Objective

Create reliable accountability and detection coverage for model-registry abuse by ensuring registry activity is recorded, forwarded, and reviewable.

## Rationale

Registry compromises often manifest first as unusual publish, pull, deletion, or permission-change activity. Without complete audit logs and forwarding, malicious model substitution, unauthorized access, and registry tampering can occur without timely detection or forensic traceability.

## Scope

Applies to all model registry deployments on Kubernetes, including Harbor, MLflow Model Registry, and on-premises Hugging Face Hub instances. Covers registry audit events, forwarding to centralized monitoring, and operational review of anomalous registry activity. Excludes artifact-retention lifecycle governance and cross-cluster replication security.

## Enforcement

- Primary Enforcement Point: Registry audit log configuration and forwarding pipeline into centralized monitoring or SIEM.
- Implementation Surface: Registry audit log enablement, event export or webhook forwarding, log ingestion verification, alerting for sensitive registry actions, and periodic review of anomalous registry operations.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Platform Engineering
