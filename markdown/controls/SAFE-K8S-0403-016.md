# SAFE-K8S-0403-016 - Environment variable secret injection prohibition and exception governance

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.3 - Secrets Management
- Maturity: Foundational
- Classification: baseline

## Statement

Organizations must prohibit injection of secrets via environment variables for production workloads by enforcing admission policies that detect and reject Pod specifications containing environment variable references to Secret objects, and must maintain an exception registry for any approved deviations with documented compensating controls and mandatory re-review dates.

## Objective

Prevent secrets from being exposed through environment variable injection, which leaks credentials into process listings, container inspection output, error logs, and child processes, by enforcing a production-wide prohibition with governed exceptions.

## Rationale

Environment variable injection is the most common but least secure secret delivery pattern. Secrets injected into environment variables are exposed in process environments, inherited by child processes, and often surface in debug output or container inspection results. Admission enforcement and exception governance are distinct from selecting approved alternative injection patterns and should be audited independently.

## Scope

Applies to all production workloads in Kubernetes clusters. Covers admission enforcement for environment variable Secret references, exception registry governance for approved deviations, and enforcement-mode configuration by namespace or environment. Excludes selection of approved alternative injection patterns, which is covered by SAFE-K8S-0403-003.

## Enforcement

- Primary Enforcement Point: Admission Enforcement for Secret Reference Patterns
- Implementation Surface: Kyverno or OPA Gatekeeper policies rejecting environment variable Secret references, namespace enforcement configuration, and documented exception governance with compensating controls.
- Owner Primary: Application Development
- Owner Supporting: Security Engineering; Platform Engineering
