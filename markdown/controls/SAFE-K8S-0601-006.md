# SAFE-K8S-0601-006 - Approved minimal base images and multi-stage build hardening

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.1 - Container Image and Registry Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

All container images deployed to production clusters must be built from approved minimal base images, must use multi-stage builds to exclude build-time tooling from runtime images, and must remove unnecessary packages and utilities before promotion.

## Objective

Reduce container attack surface by enforcing minimal approved bases and build practices that strip non-runtime content from production images.

## Rationale

General-purpose image minimization is an independent control claim: approved minimal bases and multi-stage builds prevent compilers, shells, package managers, and unused libraries from reaching runtime images. This reduces exploitable surface even before vulnerability scanning or runtime hardening controls are applied.

## Scope

Applies to all container images built or deployed in production and pre-production Kubernetes clusters, regardless of workload type.

## Enforcement

- Primary Enforcement Point: CI/CD Build Stage
- Implementation Surface: Dockerfiles, OCI build configurations, base image allowlists, and image build policies in CI/CD pipelines.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering
