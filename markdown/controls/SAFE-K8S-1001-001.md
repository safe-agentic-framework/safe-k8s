# SAFE-K8S-1001-001 - Kubernetes audit policy coverage for AI-relevant API events

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.1 - Logging and Audit
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Production Kubernetes clusters running AI workloads must configure kube-apiserver audit policies that capture AI-relevant API requests, including AI-specific CRD operations, GPU-related resource changes, admission decisions, and authorization-sensitive actions.

## Objective

Ensure that security-relevant API requests involving AI workloads are captured in the Kubernetes audit stream so they can support detection, investigation, and compliance workflows.

## Rationale

Kubernetes audit logging is generated by the kube-apiserver and records API requests and their stages. AI workloads introduce CRD types and resource patterns not covered by default audit policies. Without explicit audit policy rules for these resources, organizations lack visibility into unauthorized model deployments, GPU resource hijacking, and training job manipulation.

## Scope

Applies to production Kubernetes clusters hosting AI workloads and to non-production AI environments where security investigation or compliance evidence is required. Managed Kubernetes services remain in scope where customers control workload-level audit coverage decisions.

## Enforcement

- Primary Enforcement Point: API server audit policy rule definitions for AI-relevant resources.
- Implementation Surface: Kube-apiserver audit policy YAML, AI-specific CRD rule definitions for serving and training APIs, resource coverage for GPU-related changes, and explicit inclusion of admission and authorization-sensitive resources.
- Owner Primary: Security Operations
- Owner Supporting: Platform Engineering; AI/ML Engineering
