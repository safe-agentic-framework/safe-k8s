# SAFE-K8S-0103-009 - Cloud controller-manager IAM scoping and credential rotation

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters using a cloud controller-manager must grant only the minimum cloud IAM permissions required for cloud provider operations and must enforce automated rotation or short-lived issuance for all cloud credentials used by the cloud controller-manager.

## Objective

Reduce cloud-environment blast radius by ensuring the cloud controller-manager operates with least-privilege cloud permissions and non-persistent credentials.

## Rationale

The cloud controller-manager can manipulate high-impact infrastructure such as load balancers, routes, and volumes. Excessive IAM permissions or long-lived credentials allow a cluster compromise to expand into the broader cloud account. This control isolates the authorization and credential-lifecycle obligation from deployment topology so permission scope, workload identity, and rotation can be audited directly.

## Scope

Applies to production clusters on cloud providers with cloud controller-manager integrations. Covers cloud IAM policy scope, workload identity or equivalent federation, short-lived credentials, and periodic access review.

## Enforcement

- Primary Enforcement Point: Cloud IAM / Credential Lifecycle
- Implementation Surface: Cloud IAM policy definitions, workload identity or equivalent credential configuration, rotation automation, and IAM access review evidence.
- Owner Primary: Platform Engineering
- Owner Supporting: Cloud Platform Engineering; Security Engineering
