# SAFE-K8S-0503-010 - DNS resolution path enforcement and namespace-scoped service discovery

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.3 - Ingress, Egress, and DNS Hardening
- Maturity: Practitioner
- Classification: baseline

## Statement

DNS policy enforcement must prevent workloads from using unauthorized DNS resolution paths, and DNS-based service discovery must be limited to authorized namespaces and approved service scopes.

## Objective

Constrain how workloads resolve names and discover services so DNS cannot be used to bypass network controls or expose cross-namespace service information.

## Rationale

Even a hardened DNS infrastructure becomes a lateral-movement and policy-bypass channel if workloads can resolve names through unauthorized paths or enumerate services outside their intended namespace scope. Resolution-path enforcement and service-discovery scoping are auditable controls separate from CoreDNS configuration hardening.

## Scope

Applies to DNS egress restrictions, CNI DNS-aware policies, namespace-scoped service discovery controls, and any external DNS proxy or enforcement layer used by production workloads.

## Enforcement

- Primary Enforcement Point: DNS Policy / Service Discovery
- Implementation Surface: DNS-aware network policies, egress restrictions to CoreDNS, and service discovery scoping controls.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering
