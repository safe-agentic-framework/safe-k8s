# SAFE-K8S-0103-004 - Leader election security and control-plane high availability

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters must configure leader election for the controller-manager and scheduler with appropriate lease durations and renewal intervals, restrict RBAC permissions for leader election lease objects to control-plane components only, and deploy control-plane components in a highly available configuration with a minimum of three replicas across failure domains.

## Objective

Ensure control-plane availability and prevent unauthorized leader election manipulation by securing the leader election mechanism and deploying control-plane components across multiple failure domains.

## Rationale

Leader election determines which controller-manager and scheduler instance is actively reconciling cluster state. An attacker who can manipulate leader election lease objects can cause control-plane disruption by forcing constant leader transitions, or can install a malicious process as the active leader. Without high-availability deployment, a single control-plane node failure causes loss of scheduling, reconciliation, and security-critical controller functions.

## Scope

Applies to all production clusters. Covers leader election configuration for kube-controller-manager, kube-scheduler, and cloud-controller- manager, including lease object RBAC, lease duration tuning, and multi-replica deployment topology. For managed Kubernetes services, applies to verifying provider high-availability guarantees and monitoring control-plane availability SLOs.

## Enforcement

- Primary Enforcement Point: RBAC / Authorization
- Implementation Surface: Control-plane component deployment manifests, leader election flag configuration, RBAC policies for lease objects in kube-system namespace, and cluster topology configuration across availability zones.
- Owner Primary: Platform Engineering
- Owner Supporting: Site Reliability Engineering; Security Engineering
