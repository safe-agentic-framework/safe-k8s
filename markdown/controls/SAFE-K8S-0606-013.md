# SAFE-K8S-0606-013 - GitOps repository access, commit signing, and branch protection

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

GitOps deployment repositories must enforce repository access controls with least-privilege service accounts, require signed commits on deployment branches, and enforce branch protection rules including pull request approval and status checks.

## Objective

Ensure the integrity of Git-based deployment repositories by enforcing least-privilege access, signed changes, and protected review gates before changes can reach reconciled environments.

## Rationale

Without repository-level hardening, adversaries with repository access can inject malicious manifests through unsigned commits, bypass review through force-pushes, or abuse over-privileged GitOps credentials. Access restriction, signed commits, and branch protection protect the Git-based source of truth before reconciliation occurs.

## Scope

Applies to all Git repositories used as deployment sources for Flux or Argo CD in production and staging environments. Covers repository access controls, commit signing enforcement, branch protection, and least-privilege GitOps repository credentials.

## Enforcement

- Primary Enforcement Point: Git repository platform settings and GitOps repository credentials.
- Implementation Surface: GitHub, GitLab, or Bitbucket repository access controls; branch protection on deployment branches; commit signature verification for GitOps repositories; and Flux or Argo CD repository credentials limited to required read paths and actions.
- Owner Primary: Platform Engineering
- Owner Supporting: DevSecOps / Release Engineering; Security Engineering
