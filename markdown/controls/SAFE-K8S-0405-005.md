# SAFE-K8S-0405-005 - Credential management and multi-factor authentication

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must implement credential management best practices including credential complexity requirements, secure credential storage, and multi-factor authentication for all privileged access to Kubernetes clusters and AI infrastructure.

## Objective

Ensure that authentication credentials meet security standards and that privileged access requires multi-factor authentication, reducing the risk of credential-based initial access.

## Rationale

Weak or poorly managed credentials are the most common initial access vector for cluster compromise. Without MFA enforcement for privileged access, a single compromised password grants full administrative control. Strong credential policies combined with MFA create defense in depth for authentication.

## Scope

Applies to all authentication pathways for Kubernetes clusters including API server authentication, OIDC provider endpoints, and cloud provider authentication integrations. Covers credential policies and MFA enforcement for privileged access.

## Enforcement

- Primary Enforcement Point: OIDC provider credential and MFA policies, cloud provider IAM policies, and cluster authentication configuration.
- Implementation Surface: OIDC provider MFA enforcement policies, cloud provider IAM credential requirements, API server authentication configuration, certificate-based authentication policies, and break-glass fallback authentication procedures for OIDC provider unavailability.
- Owner Primary: Identity and Access Management
- Owner Supporting: Platform Engineering; Security Engineering
