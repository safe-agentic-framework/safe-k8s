# SAFE-K8S-0904-002 - Pipeline artifact encryption and access controls

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.4 - AI Pipeline Orchestration and Experimentation Security
- Maturity: Practitioner
- Classification: ai-specific
- Superseded By: SAFE-K8S-0904-013

## Statement

Pipeline artifacts (intermediate data, model artifacts, metadata) must be stored with encryption at rest and access controls in object storage, with per-pipeline-stage service accounts scoped to minimum required resources and retention policies aligned with data classification requirements.

## Objective

Protect ML pipeline artifacts from unauthorized access through storage encryption and least-privilege access controls.

## Rationale

ML pipelines produce intermediate artifacts containing training data subsets, feature vectors, model weights, and evaluation metrics. Without storage encryption and access controls, these artifacts are accessible to any pod with storage access.

## Scope

Applies to all artifact storage used by ML pipelines on Kubernetes, including MinIO, S3, GCS, and persistent volumes. Covers intermediate pipeline outputs, model artifacts, and pipeline metadata. Excludes stage isolation and cross-classification authorization covered by SAFE-K8S-0904-006.

## Enforcement

- Primary Enforcement Point: Object storage bucket policies and IAM configuration for pipeline artifact stores.
- Implementation Surface: Encrypted object storage (S3, GCS, MinIO) with per-namespace or per-pipeline bucket policies, object versioning for audit traceability, per-pipeline-stage service accounts scoped to minimum required resources, and retention policies aligned with data classification requirements.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Data Engineering; Security Engineering
