# SAFE-K8S-0606-003 - CI pipeline security gates

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CI pipelines must integrate mandatory security gates including SAST, DAST, and SCA scanning, secret scanning with push protection to prevent credentials from entering repositories, dependency review at pull request time, and code freshness enforcement ensuring deployed artifacts do not exceed a maximum age from committed code (build horizon). OpenSSF Scorecard or equivalent must be used for automated source code management security posture assessment. Pipelines must not promote artifacts that fail any mandatory security gate.

## Objective

Detect security defects, leaked credentials, vulnerable dependencies, and stale artifacts before they reach production by enforcing mandatory security gates in the CI pipeline.

## Rationale

Without mandatory security gates, vulnerabilities, hardcoded secrets, and vulnerable dependencies flow undetected into production. Push protection for secrets prevents credential leaks that are difficult to remediate once committed to version control history. Build horizon enforcement prevents running stale artifacts that may contain unpatched vulnerabilities. Without automated SCM security assessment, repository misconfigurations (missing branch protection, unsigned commits) weaken the foundation on which pipeline trust is built.

## Scope

Applies to all CI pipelines that produce deployable artifacts for Kubernetes clusters. SAST, SCA, and secret scanning are mandatory for all pipelines. DAST is required for pipelines producing externally-facing services. OpenSSF Scorecard applies to all source repositories. Build horizon policy applies to all production deployments.

## Enforcement

- Primary Enforcement Point: CI pipeline gate configuration, source code management platform settings, and artifact promotion policy.
- Implementation Surface: CI pipeline SAST integration (Semgrep, CodeQL, SonarQube), SCA scanning (Dependabot, Snyk, Trivy), DAST scanning on staging (OWASP ZAP, Nuclei), secret scanning with push protection (GitHub push protection, GitLeaks, TruffleHog), dependency review bots (GitHub dependency review, Renovate), build horizon enforcement comparing SLSA provenance build timestamps against deployment time (e.g., 30-day maximum artifact age), OpenSSF Scorecard for automated SCM security posture assessment, and pipeline gate failure enforcement blocking artifact promotion on any mandatory gate failure.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Development Teams
