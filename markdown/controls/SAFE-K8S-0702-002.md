# SAFE-K8S-0702-002 - Resource quotas per namespace for GPU and AI workloads

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must configure ResourceQuota objects on every namespace hosting AI workloads to enforce limits on CPU, memory, storage, GPU device count, and object counts, and must configure LimitRange objects in GPU and AI namespaces to set default and maximum per-container resource requests and limits to prevent resource monopolization by any single tenant or workload.

## Objective

Prevent any single namespace from monopolizing cluster resources, especially scarce GPU devices, by enforcing per-namespace quotas and per-container resource boundaries.

## Rationale

GPU resources are scarce and expensive. Without ResourceQuota enforcement, a single training job or a misconfigured deployment can consume all available GPUs, starving other teams' inference workloads and training pipelines. Without LimitRange defaults, pods submitted without resource requests bypass quota enforcement entirely. Object count limits prevent denial-of-service through excessive API object creation.

## Scope

Applies to all namespaces in clusters with GPU nodes or dedicated AI workloads. ResourceQuota must include nvidia.com/gpu (or equivalent device plugin resource), CPU, memory, storage, and object counts. LimitRange must define default requests and limits for containers in AI namespaces. System namespaces (kube-system) may have adjusted quotas with documented justification.

## Enforcement

- Primary Enforcement Point: ResourceQuota and LimitRange objects per namespace, enforced by the Kubernetes resource quota admission controller and validated by periodic configuration audits.
- Implementation Surface: ResourceQuota objects specifying requests.cpu, requests.memory, limits.cpu, limits.memory, requests.nvidia.com/gpu, requests.storage, persistentvolumeclaims, pods, and services hard limits per namespace, ResourceQuota scopeSelectors for BestEffort vs. non-BestEffort and PriorityClass-based differentiation, GPU device plugin resource advertisement (nvidia.com/gpu), quota utilization monitoring metrics, and integration with cost monitoring tools (OpenCost, Kubecost).
- Owner Primary: Platform Engineering
- Owner Supporting: ML Engineering; FinOps
