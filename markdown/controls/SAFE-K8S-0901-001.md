# SAFE-K8S-0901-001 - Multi-node training worker authentication and encrypted communication

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.1 - Distributed Training Workload Security
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Multi-node distributed training jobs (PyTorchJob, MPIJob, TFJob) must enforce mutual authentication between all worker nodes and must protect inter-worker gradient communication using encrypted transport where the communication framework supports it (e.g., NCCL NCCL_NET_ENCRYPTION, MPI over TLS, TensorFlow gRPC TLS).

## Objective

Ensure that inter-worker communication during distributed training is authenticated and confidentiality-protected, preventing unauthorized nodes from joining training clusters and protecting gradient data from interception.

## Rationale

Distributed training jobs exchange model gradients across multiple worker nodes over the cluster network. Without inter-worker authentication, an adversary can inject a rogue worker to steal gradient data or inject poisoned updates. Without confidentiality protection, gradient traffic is exposed to eavesdropping, enabling model reconstruction or intellectual property theft.

## Scope

Applies to all multi-node distributed training workloads running on Kubernetes, including PyTorchJob, MPIJob, and TFJob custom resources. Covers NCCL-based GPU-to-GPU communication, MPI-based parameter exchange, and high-bandwidth interconnects (NVLink, InfiniBand). Single-node training jobs are excluded unless they communicate with external parameter servers.

## Enforcement

- Primary Enforcement Point: Training job custom resource definitions and training operator configuration for worker authentication and transport encryption settings.
- Implementation Surface: Training job custom resource definitions (PyTorchJob, MPIJob, TFJob), Kubeflow Training Operator configuration, NCCL environment variables (NCCL_NET_ENCRYPTION), MPI transport encryption (Open MPI SSL support, MPICH channel encryption, or mTLS-wrapped transports), TensorFlow gRPC TLS (grpc.ssl_channel_credentials, TF_GRPC_SSL_* environment variables), and inter-worker mTLS certificate distribution.
- Owner Primary: ML Platform Engineering
- Owner Supporting: Security Engineering; Network Engineering
