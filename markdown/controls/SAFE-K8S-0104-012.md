# SAFE-K8S-0104-012 - Kubernetes supported version window and component skew compliance

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.4 - CIS Benchmarks and Patch Management
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must run a supported Kubernetes version within the project's active support window and must maintain component versions within the Kubernetes version skew policy across the API server, kubelet, controller-manager, scheduler, and administrative clients.

## Objective

Keep clusters inside supported and compatible version boundaries so security patches remain available and component interoperability does not fail under version drift.

## Rationale

Unsupported versions stop receiving security patches, and unsupported skew between cluster components can create API incompatibilities, unstable behavior, and control failures. Version currency and skew compliance are separate audit claims from the mechanics of planning and executing upgrades.

## Scope

Applies to all production clusters. Covers Kubernetes control-plane and worker node component versions, client tool versions, and managed-service version selections subject to provider support windows.

## Enforcement

- Primary Enforcement Point: API Server / Admission
- Implementation Surface: Cluster version inventory, skew compliance checks, and support-window tracking integrated into platform governance.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Change Management
