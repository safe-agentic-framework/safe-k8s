# SAFE-K8S-0405-011 - Credential policy and secure storage governance

- Domain: D04 - Identity, Access, and Secrets Management
- Knowledge Area: 4.5 - Identity Abuse Detection and Mitigation
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must enforce credential management best practices including credential complexity requirements, secure credential storage, and credential lifecycle policy controls for Kubernetes cluster and AI infrastructure authentication pathways.

## Objective

Ensure authentication credentials meet baseline security standards for strength, storage, and lifecycle handling so weak or poorly protected credentials do not become the initial access path into the cluster.

## Rationale

Weak or poorly protected credentials remain one of the most common initial access vectors for cluster compromise. Strong credential policy and secure storage reduce password guessing, credential theft, and reuse risk before interactive authentication even begins.

## Scope

Applies to authentication pathways for Kubernetes clusters, OIDC integrations, and cloud provider IAM integrations that rely on managed credentials. Covers credential policy, secure storage, and lifecycle handling requirements.

## Enforcement

- Primary Enforcement Point: Credential policy and storage controls in identity providers, cloud IAM, and cluster authentication systems.
- Implementation Surface: Credential complexity and storage policies in OIDC providers and cloud IAM, password and secret storage controls, certificate-based authentication policy baselines, and administrative credential lifecycle standards.
- Owner Primary: Identity and Access Management
- Owner Supporting: Platform Engineering; Security Engineering
