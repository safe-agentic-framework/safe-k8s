# SAFE-K8S-0101-009 - API server audit log backend durability and retention

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.1 - Kubernetes API Server Security
- Maturity: Foundational
- Classification: baseline

## Statement

Production Kubernetes clusters must direct API server audit logs to a durable backend, either a rotated log file or an approved webhook destination such as a SIEM, and must retain those logs for at least the minimum period defined by organizational policy.

## Objective

Ensure that API server audit records remain available, tamper-resistant, and searchable long enough to support investigations, compliance evidence, and anomaly detection.

## Rationale

A complete audit policy is not sufficient if logs are lost during restart, disk pressure, or backend failure. This control isolates the storage, transport, and retention obligation from audit event selection so operators can independently verify durability and log lifecycle requirements.

## Scope

Applies to all production Kubernetes clusters. Covers kube-apiserver audit log path, rotation settings, webhook export configuration, downstream aggregation, and retention enforcement.

## Enforcement

- Primary Enforcement Point: Audit Log Pipeline / Storage
- Implementation Surface: API server manifest or managed service audit export settings, log rotation configuration, webhook destination configuration, and downstream SIEM or log store retention settings.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; Compliance
