# SAFE-K8S-0606-015 - GitOps reconciliation monitoring, drift detection, and automated resync

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

GitOps controllers must verify reconciliation integrity, detect runtime configuration deviations from Git-declared state, and automatically resync or notify on drift in production environments.

## Objective

Detect divergence between declared and running cluster state and ensure unauthorized runtime changes are surfaced and remediated through GitOps reconciliation behavior.

## Rationale

If reconciliation health and drift monitoring are weak, runtime modifications can persist undetected and clusters can silently diverge from approved Git state. Automated resync and alerting are distinct operational controls focused on detecting and correcting drift after it occurs.

## Scope

Applies to production Kubernetes clusters managed through GitOps. Covers reconciliation health, drift detection, and automated correction or notification when deployed state diverges from approved Git state.

## Enforcement

- Primary Enforcement Point: Flux or Argo CD reconciliation configuration and drift monitoring telemetry.
- Implementation Surface: Argo CD auto-sync and self-heal settings, Flux reconcile intervals and health status, controller-generated drift alerts, and automated resync or notification records for unauthorized runtime changes.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; SRE / Operations
