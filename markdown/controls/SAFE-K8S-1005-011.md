# SAFE-K8S-1005-011 - Kubernetes forensic evidence acquisition and chain-of-custody procedures

- Domain: D10 - Observability, Incident Response, and Governance
- Knowledge Area: 10.5 - Incident Response for Kubernetes
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must establish forensic evidence collection procedures for Kubernetes incidents, including container or node state capture, audit log preservation, network flow capture, and documented chain-of-custody procedures for collected evidence.

## Objective

Preserve complete and admissible forensic evidence from Kubernetes incidents through documented acquisition procedures and defensible chain-of-custody handling.

## Rationale

Kubernetes evidence is highly ephemeral: pod filesystems disappear, node state changes quickly, and network flows are not retained by default. Without prepared acquisition procedures and custody documentation, responders can lose critical evidence or fail to preserve it in a form suitable for investigation, legal review, or regulator scrutiny.

## Scope

Applies to Kubernetes clusters hosting AI workloads where forensic readiness is required. Covers container-level, node-level, audit-log, and network-level evidence collection, along with custody and storage procedures for collected forensic artifacts.

## Enforcement

- Primary Enforcement Point: Forensic toolkit deployment, evidence acquisition procedures, and chain-of-custody workflow for Kubernetes infrastructure evidence.
- Implementation Surface: Container and node snapshot tooling, immutable audit log export, network flow capture, metadata preservation commands, evidence repository controls, and custody documentation templates used during Kubernetes incident response.
- Owner Primary: Security Operations
- Owner Supporting: Digital Forensics; AI/ML Engineering; Legal
