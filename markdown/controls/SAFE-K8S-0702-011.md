# SAFE-K8S-0702-011 - Isolation guarantee validation and residual risk documentation for virtual clusters

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Advanced
- Classification: baseline

## Statement

Organizations using virtual cluster technologies for tenant isolation must document the isolation guarantees and residual risks of the chosen approach and must validate tenant isolation through periodic testing and host-cluster access reviews.

## Objective

Ensure virtual-cluster-based tenant isolation is understood, documented, and periodically validated rather than assumed.

## Rationale

Virtual cluster technologies improve isolation, but their guarantees depend on architecture choices, host-cluster protections, and tenant privilege boundaries. Residual risk documentation and periodic validation are necessary to confirm that API-server separation, host cluster access restrictions, and tenant boundary assumptions still hold over time.

## Scope

Applies to multi-tenant Kubernetes environments that rely on virtual cluster or equivalent tenant control-plane separation. Documentation and validation must cover isolation guarantees, residual risks, host API access restrictions, and test evidence for tenant boundary enforcement.

## Enforcement

- Primary Enforcement Point: Residual risk documentation process, periodic isolation validation tests, and RBAC or access review of tenant access to the host cluster API server.
- Implementation Surface: Isolation model documentation, residual risk assessments, penetration tests or isolation validation exercises, RBAC audits confirming tenants cannot directly access the host cluster API server, and periodic review of isolation guarantees for the chosen virtual cluster approach.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; Cloud Platform Engineering
