# SAFE-K8S-0702-006 - Data type classification labels for pipeline data flows

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.2 - Namespace Isolation and Multi-Tenancy
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must label pipeline data flows with data type classification metadata (e.g., PII, PHI, financial, proprietary, public) at each stage of data processing, and must use these labels to enforce data handling policies through network policies, storage access controls, and admission control rules.

## Objective

Ensure that data classification is consistently tracked and enforced throughout pipeline data flows, enabling automated policy enforcement based on data sensitivity.

## Rationale

Data pipelines often process data of varying sensitivity levels across multiple stages. Without classification labels on data flow resources (PVCs, ConfigMaps, pods), sensitive data such as PII or PHI can flow into unprotected storage, cross network boundaries to untrusted namespaces, or be processed by workloads without adequate security controls. Classification labels enable automated enforcement of data handling policies.

## Scope

Applies to all data pipeline workloads, including ETL jobs, data preprocessing steps, feature engineering pipelines, and model serving data paths. Covers labels on PersistentVolumeClaims, pods, jobs, and services involved in data movement. The classification taxonomy must align with the organization's data governance framework.

## Enforcement

- Primary Enforcement Point: Admission control policies validating data classification labels on pipeline resources, network policies restricting data flow based on classification labels, and storage access controls aligned to data sensitivity levels.
- Implementation Surface: PersistentVolumeClaim metadata labels for data type classification (PII, PHI, financial, proprietary, public), pipeline orchestrator label propagation (Kubeflow Pipelines, Argo Workflows) from source PVCs to output PVCs, admission control policies (Kyverno, OPA/Gatekeeper) enforcing that pods mounting Confidential/Restricted PVCs carry corresponding classification labels, egress NetworkPolicy rules restricting high-sensitivity pods, and organizational data classification taxonomy documentation.
- Owner Primary: Data Governance
- Owner Supporting: Platform Engineering; Security Engineering
