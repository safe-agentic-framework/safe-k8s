# SAFE-K8S-0502-006 - Kubernetes CNI IPAM capacity sizing and exhaustion monitoring

- Domain: D05 - Network Security and Communication
- Knowledge Area: 5.2 - CNI Plugins and Pod Networking Security
- Maturity: Practitioner
- Classification: baseline

## Statement

IP address management for Kubernetes networking must be configured with adequate pod, service, and node CIDR capacity and must be monitored to detect impending IP pool exhaustion before scheduling failures or service outages occur.

## Objective

Maintain IP address availability for workload scheduling and service operation by ensuring CIDR allocations are sized correctly and exhaustion is detected before availability is impacted.

## Rationale

IP exhaustion in pod, service, or node CIDR ranges causes scheduling failures and service outages, which can be exploited as a denial-of-service vector. Separating IPAM sizing and monitoring from anti-spoofing controls allows capacity governance and exhaustion response to be independently audited.

## Scope

Applies to all Kubernetes clusters. Covers pod CIDR, service CIDR, and node CIDR range sizing, utilization monitoring, exhaustion thresholds, and alerting for IP address pool depletion.

## Enforcement

- Primary Enforcement Point: CNI plugin IPAM capacity management.
- Implementation Surface: CNI plugin IPAM configuration for pod, service, and node CIDR allocation, IP pool utilization monitoring and alerting thresholds, and operational review of remaining capacity.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Operations
