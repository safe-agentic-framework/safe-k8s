# SAFE-K8S-0103-003 - Cloud controller-manager isolation and credential management

- Domain: D01 - Control Plane and Cluster Hardening
- Knowledge Area: 1.3 - Controller-Manager, Scheduler, and Cloud Controller Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Production Kubernetes clusters using cloud controller-managers must deploy them separately from the core controller-manager, grant minimal cloud IAM permissions scoped only to required cloud provider operations, and enforce automated credential rotation for all cloud provider credentials used by the cloud controller-manager.

## Objective

Limit the blast radius of a cloud controller-manager compromise by isolating it from core controllers, enforcing least-privilege cloud IAM permissions, and ensuring cloud credentials are short-lived and automatically rotated.

## Rationale

The cloud controller-manager holds cloud provider credentials that can create, modify, and delete cloud infrastructure resources such as load balancers, routes, and volumes. If co-deployed with the core controller-manager, a compromise of either component exposes both Kubernetes control-plane and cloud provider credentials. Overly broad IAM permissions allow an attacker to pivot from the cluster to the broader cloud environment. Static, long-lived cloud credentials increase the window of exploitation.

## Scope

Applies to all production clusters running on cloud providers (AWS, GCP, Azure, or other providers with cloud controller-manager integrations). Covers deployment isolation, IAM policy scoping, and credential lifecycle management. Does not apply to bare-metal or on-premises clusters without cloud provider integration.

## Enforcement

- Primary Enforcement Point: RBAC / Authorization
- Implementation Surface: Cloud controller-manager deployment manifests, cloud IAM policy definitions, credential rotation automation (workload identity, IRSA, or equivalent), and periodic IAM access review.
- Owner Primary: Platform Engineering
- Owner Supporting: Cloud Platform Engineering; Security Engineering
