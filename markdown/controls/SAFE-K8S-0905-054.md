# SAFE-K8S-0905-054 - External model source NetworkPolicy enforcement for protected namespaces

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.5 - AI Supply Chain and Model Lifecycle
- Maturity: Advanced
- Classification: ai-specific

## Statement

Organizations must enforce the approved external model source allowlist through NetworkPolicy or equivalent egress controls that block direct pulls from public hubs to production or otherwise protected namespaces.

## Objective

Prevent protected workloads from bypassing approved-source policy by restricting runtime network access to unapproved external model sources.

## Rationale

Even with an approved-source policy, production workloads can still reach public model hubs unless network egress is explicitly constrained. Runtime egress restriction is required to stop direct pulls from unapproved external sources.

## Scope

Applies to production and protected Kubernetes namespaces that consume external models. Covers NetworkPolicy or equivalent egress controls restricting direct access to public model hubs and other unapproved external model endpoints. Excludes admission-time validation of model source references.

## Enforcement

- Primary Enforcement Point: NetworkPolicy and egress control enforcement point.
- Implementation Surface: NetworkPolicy blocking direct public hub access from protected namespaces, CNI or cloud egress controls restricting unapproved model source endpoints, and enforcement telemetry showing denied runtime access attempts.
- Owner Primary: Security Engineering
- Owner Supporting: ML Platform Engineering; Governance
