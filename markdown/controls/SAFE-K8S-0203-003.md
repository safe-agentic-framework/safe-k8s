# SAFE-K8S-0203-003 - Cloud metadata service protection and IMDSv2 enforcement

- Domain: D02 - Node, Runtime, and OS Security
- Knowledge Area: 2.3 - Host OS and Kernel Hardening
- Maturity: Foundational
- Classification: baseline

## Statement

Kubernetes nodes running on cloud infrastructure must enforce IMDSv2 (Instance Metadata Service v2) with hop-limit restrictions, and must implement network-level controls to prevent pod workloads from directly accessing the cloud provider metadata endpoint (169.254.169.254).

## Objective

Prevent container workloads from accessing cloud instance metadata credentials, which could enable lateral movement to cloud APIs and privilege escalation beyond the Kubernetes cluster boundary.

## Rationale

The cloud metadata service (169.254.169.254) provides instance-level credentials, network configuration, and user data to any process that can reach the endpoint. Without IMDSv2 enforcement, SSRF vulnerabilities in containerized applications can retrieve IAM role credentials via simple HTTP GET requests. These credentials often carry broad cloud permissions enabling access to storage, databases, and other cloud resources outside the cluster. IMDSv2 with reduced hop limits and network-level blocking together prevent this common cloud-native attack vector.

## Scope

Applies to all Kubernetes nodes running on AWS, GCP, Azure, or other cloud providers that expose instance metadata services. On-premises clusters without cloud metadata services are exempt. Managed Kubernetes services must configure available metadata protection options (e.g., AWS node group IMDSv2 requirement, GKE Workload Identity).

## Enforcement

- Primary Enforcement Point: Cloud infrastructure provisioning templates (Terraform, CloudFormation) and node-level network policies or iptables rules blocking metadata endpoint access from pod CIDR ranges.
- Implementation Surface: Cloud infrastructure provisioning templates (Terraform, CloudFormation), IMDSv2 enforcement configuration (AWS), GKE metadata server and Workload Identity (GCP), Azure Workload Identity (Azure), NetworkPolicy egress rules blocking metadata endpoints, and workload identity federation configuration.
- Owner Primary: Cloud Infrastructure
- Owner Supporting: Platform Engineering; Security Engineering
