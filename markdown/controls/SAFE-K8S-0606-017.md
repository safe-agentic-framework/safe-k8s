# SAFE-K8S-0606-017 - CNCF lifecycle phase security coverage

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

CI/CD pipelines must apply required security controls across all CNCF lifecycle phases (Develop, Distribute, Deploy), with explicit phase-completion gates preventing artifact promotion or deployment until the controls for that phase have executed and passed.

## Objective

Ensure CI/CD security coverage is defined and enforced at each CNCF lifecycle phase so supply chain controls are not concentrated in only one stage of the delivery path.

## Rationale

A pipeline can implement strong point controls but still leave major lifecycle gaps if develop-time, distribute-time, or deploy-time protections are missing. Treating Develop, Distribute, and Deploy as distinct control checkpoints ensures that security coverage is complete across the software delivery path rather than assumed from a single successful gate.

## Scope

Applies to all CI/CD pipelines that build, promote, sign, attest, or deploy artifacts to Kubernetes clusters. Covers phase coverage from source and build activity through artifact distribution and deployment gating.

## Enforcement

- Primary Enforcement Point: CI/CD pipeline phase definitions, promotion gates, and deployment approval logic.
- Implementation Surface: CI/CD pipeline definitions covering Develop, Distribute, and Deploy phases, with required stage-specific controls, promotion criteria, and evidence that artifacts do not advance when a lifecycle phase gate is incomplete or failed.
- Owner Primary: DevSecOps / Release Engineering
- Owner Supporting: Security Engineering; Platform Engineering
