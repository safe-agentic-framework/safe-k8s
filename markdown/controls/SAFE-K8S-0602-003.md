# SAFE-K8S-0602-003 - Admission enforcement of image signatures

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.2 - Image Signing and Admission Enforcement
- Maturity: Practitioner
- Classification: baseline

## Statement

Kubernetes clusters must deploy an admission controller (Kyverno, OPA/Gatekeeper, or Sigstore policy-controller) configured to reject any container image that lacks a valid cryptographic signature from an authorized signing identity. The admission policy must be set to fail closed, and no bypass mechanism may exist outside of a documented and time-limited emergency break-glass procedure.

## Objective

Enforce image signature verification at the Kubernetes admission layer to prevent unsigned or tampered images from running in the cluster.

## Rationale

Signing images without enforcing verification at deployment time provides no runtime protection. An adversary who gains access to a registry or CI/CD pipeline can push unsigned malicious images that will be accepted by the cluster. Admission-time enforcement closes this gap by creating a mandatory verification checkpoint. Without fail-closed configuration, a webhook outage silently degrades the control, allowing unsigned images to deploy.

## Scope

Applies to all production and staging Kubernetes clusters. All namespaces must be covered unless explicitly exempted in a documented policy (e.g., kube-system for provider-managed components). Emergency break-glass procedures must be audited and time-limited.

## Enforcement

- Primary Enforcement Point: Kubernetes validating admission webhook (Kyverno, OPA/Gatekeeper, or Sigstore policy-controller).
- Implementation Surface: Kubernetes ValidatingWebhookConfiguration with failurePolicy: Fail, admission controller deployments (Kyverno verifyImages rules, OPA/Gatekeeper constraint templates, Sigstore policy-controller), image signature verification policies matching signing identities to approved CI pipeline OIDC subjects, break-glass annotation mechanisms (time-limited bypass annotations with auto-expiry), admission controller audit logs for rejected unsigned images and bypass events, and namespace-scoped exemption policies.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
