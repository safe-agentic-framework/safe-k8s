# SAFE-K8S-0704-007 - Unified cloud and Kubernetes audit log ingestion

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.4 - Cloud Provider Security Integration
- Maturity: Practitioner
- Classification: baseline

## Statement

Organizations must forward cloud provider audit logs and Kubernetes audit events to a common security monitoring platform, retain them according to policy, and ensure both telemetry sources are available for managed Kubernetes security operations and investigations.

## Objective

Ensure cloud and Kubernetes audit telemetry is centrally available in one monitoring plane for managed cluster security operations.

## Rationale

Managed Kubernetes investigations fail when cloud-provider and Kubernetes audit records are collected into separate systems or retained inconsistently. Centralizing both audit streams creates the shared monitoring plane required for downstream detection, triage, and forensic review.

## Scope

Applies to all managed Kubernetes clusters on cloud providers (EKS, GKE, AKS). Cloud audit logs and Kubernetes audit events must be forwarded to the same SIEM or log analytics platform, and retention must meet organizational and regulatory requirements.

## Enforcement

- Primary Enforcement Point: Cloud audit log forwarding configuration, Kubernetes audit log forwarding, unified monitoring platform onboarding, and retention enforcement.
- Implementation Surface: Cloud provider audit log services (AWS CloudTrail, GCP Cloud Audit Logs, Azure Activity Log), Kubernetes audit log pipelines, SIEM or log analysis platforms, retention controls, and data-source health monitoring for audit ingestion.
- Owner Primary: Security Engineering
- Owner Supporting: Cloud Platform Engineering; Platform Engineering; SOC
