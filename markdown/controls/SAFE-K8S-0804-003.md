# SAFE-K8S-0804-003 - Key management and sealed storage for confidential AI enclaves

- Domain: D08 - GPU, Accelerator, and Confidential Computing
- Knowledge Area: 8.4 - Confidential Computing for AI Workloads
- Maturity: Advanced
- Classification: ai-specific
- Superseded By: SAFE-K8S-0804-005

## Statement

Encryption keys protecting models and data within confidential computing enclaves must be managed through a hardware-rooted key management system, with key release conditioned on successful TEE attestation, and sealed storage must be used to bind encrypted model artifacts and data to specific attested enclave configurations.

## Objective

Ensure that cryptographic keys for model decryption and data access are only released to verified TEE environments and that model artifacts remain bound to their authorized enclave configuration, preventing key extraction or model decryption outside the trusted environment.

## Rationale

If encryption keys for models and training data are managed through standard key management systems without attestation-conditioned release, a compromised host or operator can retrieve the keys and decrypt protected assets outside the TEE. Sealed storage binds encrypted artifacts to a specific hardware and software measurement, ensuring that models encrypted for one enclave configuration cannot be decrypted if the platform configuration is modified. Without these controls, confidential computing provides memory encryption but not a complete trust chain for model protection.

## Scope

Applies to all confidential AI workloads where models or training data are encrypted at rest and require key release into TEE enclaves. Covers integration with KMS (Vault, cloud KMS, HSM), sealed storage mechanisms, and attestation-conditioned key release policies. This is an emerging control requiring vendor-specific sealed storage implementations.

## Enforcement

- Primary Enforcement Point: Key management system attestation-conditioned release policies, sealed storage configuration in enclave runtime, and model encryption and decryption workflow integration with attestation verification.
- Implementation Surface: Hardware-rooted key management systems (HashiCorp Vault with attestation plugin, cloud KMS with TEE attestation integration, hardware security modules), attestation-conditioned key release policies, TEE sealed storage mechanisms binding encrypted model artifacts to attested enclave measurements (sealing identity), key rotation procedures triggered by TEE firmware updates, escrow key backup with dual-authorization access controls, and model encryption/decryption workflow integration within enclave runtimes.
- Owner Primary: Security Engineering
- Owner Supporting: Key Management; ML Platform Engineering; GPU Infrastructure Engineering
