# SAFE-K8S-0605-005 - CRD validation and webhook hardening for AI custom resources

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.5 - Admission Control
- Maturity: Practitioner
- Classification: ai-specific

## Statement

AI-specific Custom Resource Definitions (InferenceService, TrainingJob, RayCluster, PodGroup) must have validating admission webhooks that enforce schema correctness, security constraints, and organizational policy. Webhook endpoints for AI CRDs must be hardened against abuse and denial-of-service.

## Objective

Prevent misconfigured or malicious AI custom resources from being admitted to the cluster by enforcing validation and security constraints specific to AI operator CRDs.

## Rationale

AI custom resources (InferenceService, TrainingJob, RayCluster) define workloads that consume expensive GPU resources and process sensitive data. Without CRD-specific validation webhooks, malformed or malicious custom resources can request excessive GPU allocations, reference unauthorized model sources, or bypass security constraints. Webhook hardening prevents adversaries from abusing or overwhelming CRD admission endpoints.

## Scope

Applies to all Kubernetes clusters running AI/ML operator frameworks (KServe, Kubeflow, Ray, Volcano) that install custom resource definitions. Includes production, staging, and shared development clusters with GPU node pools. Clusters not running AI workload operators may document this control as not applicable.

## Enforcement

- Primary Enforcement Point: Kubernetes validating admission webhooks for AI CRDs.
- Implementation Surface: Kubernetes ValidatingWebhookConfiguration resources for AI CRDs (InferenceService, TrainingJob, RayCluster, PodGroup), AI operator admission webhooks (KServe, Kubeflow Training Operator, KubeRay, Volcano) configured with failurePolicy: Fail, CRD structural schema validation and security constraint enforcement, webhook endpoint rate limiting and resource constraints, and RBAC integration for AI CRD access control (SAFE-K8S-0401-004).
- Owner Primary: Platform Engineering
- Owner Supporting: ML Platform Engineering; Security Engineering
