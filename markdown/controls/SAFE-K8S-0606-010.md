# SAFE-K8S-0606-010 - Infrastructure as Code security and policy-as-code integrity

- Domain: D06 - Supply Chain, Images, and Admission Control
- Knowledge Area: 6.6 - CI/CD and GitOps Pipeline Security
- Maturity: Practitioner
- Classification: baseline

## Statement

Infrastructure as Code (IaC) definitions for deployment platforms must undergo security scanning and integrity verification in the CI pipeline. Runtime settings must be defined as policy as code, and configuration code must have integrity verification preventing unauthorized modification. Policy-as-code repositories must enforce signed commits and branch protection.

## Objective

Prevent infrastructure and runtime configuration from being weakened or tampered with by applying security scanning to IaC templates and integrity verification to policy-as-code definitions.

## Rationale

IaC templates that define cluster infrastructure can be modified to weaken network policies, disable security features, or create backdoor access. Without security scanning, these misconfigurations reach production undetected. Unsigned configuration code enables unauthorized runtime setting changes that bypass GitOps controls. Policy-as-code integrity verification ensures that security policies (network policies, RBAC, pod security, admission rules) cannot be modified without authorized review.

## Scope

Applies to all Terraform/Pulumi/CloudFormation templates and policy-as-code definitions used for production and staging environments. Covers IaC templates for cluster infrastructure, network configuration, and security policy definitions.

## Enforcement

- Primary Enforcement Point: IaC scanning in CI pipeline and policy-as-code repository integrity verification.
- Implementation Surface: IaC security scanning tools (Checkov, tfsec, KICS) for Terraform/Pulumi/CloudFormation templates integrated as CI pipeline gates, policy-as-code Git repositories with signed commits and branch protection for network policies, RBAC, pod security, and admission policies, configuration code integrity verification preventing unauthorized runtime setting modifications, and CI pipeline gate enforcement blocking IaC changes that fail security scanning.
- Owner Primary: Platform Engineering
- Owner Supporting: Security Engineering; DevSecOps / Release Engineering
