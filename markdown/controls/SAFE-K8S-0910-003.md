# SAFE-K8S-0910-003 - Multi-cluster network security for distributed AI workloads

- Domain: D09 - AI Workload Security: Training, Serving, and Pipelines
- Knowledge Area: 9.10 - Multi-Cluster and Federated AI Security
- Maturity: Advanced
- Classification: ai-specific

## Statement

Cross-cluster network communication for distributed AI workloads (cross-cluster gradient exchange, multi-cluster inference routing, and cross-cluster pipeline data transfer) must use encrypted and authenticated channels, must enforce cross-cluster network policies that restrict communication to authorized workload pairs, and must monitor cross-cluster traffic for anomalous patterns.

## Objective

Protect cross-cluster AI communication from interception, tampering, and unauthorized access by extending network security controls beyond single-cluster NetworkPolicy boundaries.

## Rationale

Cross-cluster AI workloads create network paths that bypass single-cluster network policy enforcement. Gradient exchange, inference routing, and pipeline data transfer between clusters require network connectivity that standard Kubernetes NetworkPolicy cannot govern. Without cross-cluster network policy, any workload in a federated cluster can communicate with any workload in another cluster. Without cross-cluster traffic monitoring, data exfiltration through cross-cluster channels is undetectable.

## Scope

Applies to all cross-cluster network communication for AI workloads. Covers distributed training gradient exchange across clusters, multi-cluster inference request routing, and cross-cluster pipeline data transfer. Single-cluster network security is addressed by D5 controls.

## Enforcement

- Primary Enforcement Point: Multi-cluster service mesh configuration, cross-cluster network policy enforcement, VPN/WireGuard tunnel configuration, and cross-cluster traffic monitoring infrastructure.
- Implementation Surface: Multi-cluster service mesh (Istio multi-cluster, Cilium ClusterMesh, Linkerd multi-cluster) with mTLS, VPN/WireGuard tunnels with SPIFFE-based endpoint authentication, cross-cluster AuthorizationPolicy or Cilium ClusterMesh network policies (default-deny with explicit allow rules), cross-cluster traffic monitoring dashboards and anomaly detection for unexpected sources and data volumes, and bandwidth and latency quality-of-service controls prioritizing distributed training communication.
- Owner Primary: Network Engineering
- Owner Supporting: Platform Engineering; Security Engineering; ML Platform Engineering
