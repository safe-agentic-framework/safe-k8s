# SAFE-K8S-0703-020 - GPU workload abuse detection and response

- Domain: D07 - Storage, Multi-tenancy, and Resource Governance
- Knowledge Area: 7.3 - Resource Governance and Priority
- Maturity: Practitioner
- Classification: ai-specific

## Statement

Organizations must monitor GPU-equipped Kubernetes environments for cryptomining, resource squatting, personal workloads, and other unauthorized GPU usage patterns, and must investigate or terminate confirmed abusive consumption.

## Objective

Detect and respond to non-sanctioned GPU consumption patterns before they create sustained cost waste, tenant impact, or evidence of broader compromise.

## Rationale

GPU abuse often appears first as anomalous utilization, suspicious workload fingerprints, or repeated unauthorized consumption rather than as an explicit policy declaration. Without focused detection and response, cryptomining, personal workloads, and abandoned GPU reservations can persist for long periods, wasting scarce AI capacity and masking deeper security incidents.

## Scope

Applies to all GPU-equipped nodes and GPU-consuming workloads in Kubernetes clusters. Monitoring must cover GPU utilization anomalies, workload fingerprinting, and alert or incident handling for unauthorized consumption patterns.

## Enforcement

- Primary Enforcement Point: GPU telemetry monitoring, workload fingerprinting, and incident response workflows for suspected abuse.
- Implementation Surface: GPU telemetry pipelines, anomaly detection rules, workload fingerprinting logic, cryptomining signatures, and response workflows that quarantine, terminate, or investigate abusive GPU workloads.
- Owner Primary: Security Engineering
- Owner Supporting: Platform Engineering; ML Engineering
